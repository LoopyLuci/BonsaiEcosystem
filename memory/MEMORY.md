## 💡 KEY CONCEPTS

### Universal Module System (UMS)
- Enables all 2,413 crates to be discovered, loaded, and managed dynamically
- USEE searches < 5ms
- Module loading < 100ms
- Agents autonomously discover and control modules
- See: `module_system_complete.md`

### Three-Layer Architecture
- **Layer 1**: Microkernel (formal verification, 9 subsystems)
- **Layer 2**: OS services (6 services, 4 languages, 2,413 crates)
- **Layer 3**: Applications (Workspace, Buddy, Control Panel)
- See: `repository_complete_overview.md`

### Dynamic Module Loading
- Load/unload modules on-demand (< 100ms)
- Hot-reload without downtime
- Dependency resolution (parallel)
- Graceful shutdown
- See: `module_system_complete.md`

### Capability-Based Security
- Unforgeable tokens for access control
- Module signing and verification
- RBAC for fine-grained control
- Audit logging
- See: `OMNISYSTEM_SECURITY_COMPLIANCE.md`

## 🔧 OPERATIONS

### Build Commands
```bash
# Build everything
cargo build --release --all

# Build by layer
cargo build --release -p uosc              # Layer 1
cargo build --release -p omnisystem-*      # Layer 2
cargo build --release -p bonsai-*          # Layer 3
```

### Deployment Modes (6)
1. **Co-OS** - Native kernel (< 1ms latency)
2. **VM** - Virtual machines (5-20ms)
3. **Container** - Docker/Kubernetes (10-50ms)
4. **Library OS** - Embedded (< 5ms)
5. **Bare-Metal** - Raw hardware (< 1ms)
6. **Cloud** - AWS/Azure/GCP (20-100ms)

See: `deployment_and_operations.md`
