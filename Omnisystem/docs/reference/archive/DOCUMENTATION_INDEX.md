## 🚀 Quick Reference Cards

### Commands by Task

| Task | Command |
|------|---------|
| **Deploy an app** | `bonsai container deploy --blueprint app.bp` |
| **Check system health** | `bonsai container health` |
| **Ask BonsAI a question** | `bonsai chat "your question here"` |
| **Find bugs** | `bonsai sweep --full --ai` |
| **Fix bugs automatically** | `bonsai sweep --full --auto-fix` |
| **Train AI on your code** | `bonsai bonsai-v2 train --source kdb://my-code` |
| **Stream media** | `bonsai media capture && bonsai media stream` |
| **Manage via chat** | Start OmniBot on Discord/Telegram and ask naturally |
| **Monitor with Octopus** | `bonsai chat --mode octopus` |

### Glossary (Quick)

| Term | Means |
|------|-------|
| **BACE** | Atomic Compiler Engine (instant feedback + native speed) |
| **BCF** | Bonsai Container Fabric (Docker replacement) |
| **Blueprint** | Configuration file describing a system |
| **BUSH** | Universal Simulator (run any software on any hardware) |
| **Capability** | Digital permission token (signed, time-limited) |
| **Crystal** | Immutable system snapshot (safe updates) |
| **Echo** | Peer-to-peer networking (no central server) |
| **KDB/KMDB** | Knowledge database (searchable facts and procedures) |
| **Octopus AI** | Server management AI (autonomous healing) |
| **OmniBot** | Unified chat bot (control Bonsai from Telegram/Discord) |
| **Sanctum** | Isolated vault (hardware-enforced process isolation) |
| **Universe** | Immutable audit log (time-travel debugging) |
| **UOSC** | Unified Secure Operating System (the foundation) |

## 📝 Documentation Notes

### These Docs Are
✅ **Comprehensive** — Cover 100% of features  
✅ **Simple** — Explained in plain language  
✅ **Accurate** — Reflect the actual system  
✅ **Up-to-date** — Updated as Bonsai evolves  
✅ **Public** — No private systems described in detail  

### These Docs Are NOT
❌ **Marketing material** — Technical and honest  
❌ **Tutorials** (mostly) — More reference than step-by-step  
❌ **API documentation** — For that, use `bonsai --help`  
❌ **Code comments** — For that, read the source  

### Private Components NOT Documented Here
The research-only variant models and specialized deployment systems are kept completely private and are never uploaded to GitHub. The documentation focuses on public, open-source components.
