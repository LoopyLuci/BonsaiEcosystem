The foundational pattern for scalability is **write‑read decoupling** and **compute‑storage separation**. Industry surveys have identified five principal patterns: node‑level read‑write separation, compute‑storage separation, full in‑memory indexing, log‑structured write paths, and in‑place partial updates. ### 🧠 Hybrid Search & Ranking Engine

The system will combine multiple retrieval paths simultaneously, which is emerging as a robustness design in enterprise search. Each query triggers lexical search (BM25), dense vector search (via HNSW), and optionally learned sparse retrieval (LSR) in parallel. The results are merged via a **learning‑to‑rank (LTR) model** that can integrate signals from each path.     data: DashMap<K, CacheEntry<V>>,
```rust
    // Policy engine for eviction (shared across threads).
    policy: Arc<dyn EvictionPolicy<K> + Send + Sync>,
    // Maximum number of entries (capacity).
    capacity: usize,
    // Optional TTL manager.
    ttl: Option<TtlManager<K, V>>,
    // Optional secondary storage tier (e.g., disk/remote).
    tier: Option<Arc<dyn Tier<K, V>>>,
    // Statistics.
    stats: Arc<Stats>,
    // Background thread handle (for TTL cleanup).
    _cleanup_handle: Option<std::thread::JoinHandle<()>>,
}

impl<K, V> Cache<K, V>
where
    K: Hash + Eq + Clone + Send + Sync + 'static,
    V: Clone + Send + Sync + 'static,
{
    /// Create a new cache builder.
    pub fn builder() -> CacheBuilder<K, V> {
        CacheBuilder::new()
    }

    /// Insert a key‑value pair.
    pub fn insert(&self, key: K, value: V) {
        self.insert_with_ttl(key, value, None);
    }

    /// Insert with custom TTL (in seconds).
    pub fn insert_with_ttl(&self, key: K, value: V, ttl_secs: Option<u64>) {
        let now = Instant::now();
        let expiry = ttl_secs.map(|s| now + Duration::from_secs(s));
        let entry = CacheEntry::new(value, expiry);

        // Notify policy that we are accessing/adding.
        self.policy.record_access(&key);

        // Insert or update.
        let old = self.data.insert(key.clone(), entry);

        // If insertion caused size to exceed capacity, evict.
        if self.data.len() > self.capacity {
            if let Some(victim) = self.policy.evict() {
                self.data.remove(&victim);
                self.stats.inc_evictions();
            }
        }

        // Update stats.
        self.stats.inc_inserts();
        if old.is_some() {
            self.stats.inc_updates();
        }
    }

    /// Get a value from the cache.
    pub fn get(&self, key: &K) -> Option<V> {
        let now = Instant::now();
        if let Some(mut entry_ref) = self.data.get_mut(key) {
            let entry = entry_ref.value_mut();
            // Check TTL.
            if let Some(exp) = entry.expiry {
                if now >= exp {
                    // Expired – remove and count as miss.
                    drop(entry_ref);
                    self.data.remove(key);
                    self.stats.inc_expirations();
                    return None;
                }
            }
            // Record access for policy.
            self.policy.record_access(key);
            entry.last_access = now;
            self.stats.inc_hits();
            Some(entry.value.clone())
        } else {
            // Miss – try secondary tier.
            if let Some(tier) = &self.tier {
                if let Some(val) = tier.get(key) {
                    // Promote to main cache.
                    self.insert(key.clone(), val.clone());
                    self.stats.inc_tier_hits();
                    return Some(val);
                }
            }
            self.stats.inc_misses();
            None
        }
    }

    /// Remove a key from the cache.
    pub fn remove(&self, key: &K) -> Option<V> {
        if let Some((_, entry)) = self.data.remove(key) {
            self.policy.record_removal(key);
            self.stats.inc_removals();
            Some(entry.value)
        } else {
            None
        }
    }

    /// Clear the entire cache.
    pub fn clear(&self) {
        self.data.clear();
        self.policy.clear();
        self.stats.reset();
    }

    /// Get current cache size.
    pub fn len(&self) -> usize {
        self.data.len()
    }

    /// Check if cache is empty.
    pub fn is_empty(&self) -> bool {
        self.data.is_empty()
    }

    /// Retrieve statistics.
    pub fn stats(&self) -> Stats {
        self.stats.snapshot()
    }
}

// ----- Builder Pattern -----
pub struct CacheBuilder<K, V> {
    capacity: usize,
    policy: Policy,
    ttl_secs: Option<u64>,
    tier: Option<Arc<dyn Tier<K, V>>>,
    enable_metrics: bool,
}

impl<K, V> CacheBuilder<K, V>
where
    K: Hash + Eq + Clone + Send + Sync + 'static,
    V: Clone + Send + Sync + 'static,
{
    fn new() -> Self {
        Self {
            capacity: 1000,
            policy: Policy::Lru,
            ttl_secs: None,
            tier: None,
            enable_metrics: false,
        }
    }

    pub fn capacity(mut self, cap: usize) -> Self {
        self.capacity = cap;
        self
    }

    pub fn policy(mut self, pol: Policy) -> Self {
        self.policy = pol;
        self
    }

    pub fn ttl_secs(mut self, secs: u64) -> Self {
        self.ttl_secs = Some(secs);
        self
    }

    pub fn with_tier(mut self, tier: Arc<dyn Tier<K, V>>) -> Self {
        self.tier = Some(tier);
        self
    }

    pub fn enable_metrics(mut self) -> Self {
        self.enable_metrics = true;
        self
    }

    pub fn build(self) -> Cache<K, V> {
        let policy: Arc<dyn EvictionPolicy<K> + Send + Sync> = match self.policy {
            Policy::Lru => Arc::new(LruPolicy::new(self.capacity)),
            Policy::Lfu => Arc::new(LfuPolicy::new(self.capacity)),
            Policy::Arc => Arc::new(ArcPolicy::new(self.capacity)),
            Policy::TinyLfu => Arc::new(TinyLfuPolicy::new(self.capacity)),
        };

        let ttl_mgr = self.ttl_secs.map(|secs| {
            let data = Arc::clone(&policy); // Not used directly; we need the cache reference. For brevity, we omit full background TTL daemon here; but TtlManager would own a weak reference to cache.
            TtlManager::new(Duration::from_secs(secs))
        });

        // Start background TTL cleanup thread (simplified – would pass a weak cache handle).
        let _cleanup_handle = ttl_mgr.as_ref().map(|mgr| mgr.start_cleanup_thread());

        Cache {
            data: DashMap::new(),
            policy,
            capacity: self.capacity,
            ttl: ttl_mgr,
            tier: self.tier,
            stats: Arc::new(Stats::new(self.enable_metrics)),
            _cleanup_handle,
        }
    }
}

    Lru,
    Lfu,
    Arc,
    TinyLfu,
}
```

## `src/policy/lru.rs` – LRU using `crossbeam::sync::ShardedLock` + `linked_hash_map`

Because `linked_hash_map` is not concurrency‑ready, we implement a concurrent LRU using a sharded approach or a simple `parking_lot::Mutex` for demonstration. In production, you'd use `evmap` or a lock‑striped design.

```rust
use super::EvictionPolicy;
use parking_lot::Mutex;
use std::collections::VecDeque;
use std::hash::Hash;
use std::sync::atomic::{AtomicUsize, Ordering};

/// A simple thread‑safe LRU using a single Mutex (fine for moderate concurrency).
pub struct LruPolicy<K> {
    map: Mutex<linked_hash_map::LinkedHashMap<K, ()>>,
    capacity: usize,
    size: AtomicUsize,
}

impl<K: Hash + Eq + Clone> LruPolicy<K> {
    pub fn new(capacity: usize) -> Self {
        Self {
            map: Mutex::new(linked_hash_map::LinkedHashMap::new()),
            capacity,
            size: AtomicUsize::new(0),
        }
    }
}

impl<K: Hash + Eq + Clone + Send + Sync> EvictionPolicy<K> for LruPolicy<K> {
    fn record_access(&self, key: &K) {
        let mut map = self.map.lock();
        if map.contains_key(key) {
            // Move to front (most recent).
            map.remove(key);
            map.insert(key.clone(), ());
        } else {
            if map.len() >= self.capacity {
                // Evict last (LRU)
                if let Some((oldest, _)) = map.pop_front() {
                    let _ = self.size.fetch_sub(1, Ordering::Relaxed);
                }
            }
            map.insert(key.clone(), ());
            self.size.fetch_add(1, Ordering::Relaxed);
        }
    }

    fn record_removal(&self, key: &K) {
        let mut map = self.map.lock();
        if map.remove(key).is_some() {
            self.size.fetch_sub(1, Ordering::Relaxed);
        }
    }

    fn evict(&self) -> Option<K> {
        let mut map = self.map.lock();
        if let Some((k, _)) = map.pop_front() {
            self.size.fetch_sub(1, Ordering::Relaxed);
            Some(k)
        } else {
            None
        }
    }

    fn clear(&self) {
        let mut map = self.map.lock();
        map.clear();
        self.size.store(0, Ordering::Relaxed);
    }
}
```

## 1. Complete ARC Policy Implementation

ARC dynamically balances between recency (LRU) and frequency (LFU). It maintains four lists: `B1` (ghost for recency), `B2` (ghost for frequency), `T1` (recency cache), `T2` (frequency cache). The implementation below is **thread‑safe** using a `parking_lot::Mutex` – suitable for moderate concurrency. For higher throughput, a sharded approach would be needed.

### Add to `Cargo.toml`


### `src/policy/arc.rs`

```rust
use super::EvictionPolicy;
use linked_hash_map::LinkedHashMap;
use parking_lot::Mutex;
use std::collections::VecDeque;
use std::hash::Hash;
use std::sync::atomic::{AtomicUsize, Ordering};

/// Adaptive Replacement Cache (ARC) – concurrent version.
/// Uses a single Mutex; lock contention is acceptable for many use cases.
pub struct ArcPolicy<K> {
    inner: Mutex<ArcInner<K>>,
    capacity: usize,
    size: AtomicUsize,
}

struct ArcInner<K> {
    // Actual cache pages
    t1: LinkedHashMap<K, ()>, // recency (MRU)
    t2: LinkedHashMap<K, ()>, // frequency (MRU)
    // Ghost lists
    b1: VecDeque<K>, // ghost of recency
    b2: VecDeque<K>, // ghost of frequency
    // Adaptation parameters
    p: usize, // target size for T1
}

impl<K: Hash + Eq + Clone> ArcPolicy<K> {
    pub fn new(capacity: usize) -> Self {
        Self {
            inner: Mutex::new(ArcInner {
                t1: LinkedHashMap::new(),
                t2: LinkedHashMap::new(),
                b1: VecDeque::new(),
                b2: VecDeque::new(),
                p: 0,
            }),
            capacity,
            size: AtomicUsize::new(0),
        }
    }

    fn replace(&self, inner: &mut ArcInner<K>, key: &K) {
        // Called when a cache miss occurs and cache is full.
        if !inner.t1.is_empty() && (inner.t1.len() > inner.p || (inner.t1.len() == inner.p && !inner.b2.contains(key))) {
            // Evict from T1 (LRU page)
            if let Some((old, _)) = inner.t1.pop_front() {
                inner.b1.push_back(old);
                if inner.b1.len() > self.capacity {
                    inner.b1.pop_front();
                }
            }
        } else {
            // Evict from T2 (LRU page)
            if let Some((old, _)) = inner.t2.pop_front() {
                inner.b2.push_back(old);
                if inner.b2.len() > self.capacity {
                    inner.b2.pop_front();
                }
            }
        }
    }
}

impl<K: Hash + Eq + Clone + Send + Sync> EvictionPolicy<K> for ArcPolicy<K> {
    fn record_access(&self, key: &K) {
        let mut inner = self.inner.lock();

        // Case 1: Hit in T1 or T2 -> move to MRU of T2
        if inner.t1.contains_key(key) {
            inner.t1.remove(key);
            inner.t2.insert(key.clone(), ());
            return;
        }
        if inner.t2.contains_key(key) {
            // move to MRU of T2 (just reinsert)
            inner.t2.remove(key);
            inner.t2.insert(key.clone(), ());
            return;
        }

        // Case 2: Miss – we need to insert a new page.
        // First, adjust p based on ghost hit.
        if inner.b1.contains(key) {
            // Ghost hit in B1: increase p
            let delta = 1.max(inner.b2.len() / inner.b1.len());
            inner.p = (inner.p + delta).min(self.capacity);
            // Remove from B1
            inner.b1.retain(|x| x != key);
        } else if inner.b2.contains(key) {
            // Ghost hit in B2: decrease p
            let delta = 1.max(inner.b1.len() / inner.b2.len());
            inner.p = inner.p.saturating_sub(delta);
            // Remove from B2
            inner.b2.retain(|x| x != key);
        }

        // If cache is full, replace a page
        let total = inner.t1.len() + inner.t2.len();
        if total >= self.capacity {
            self.replace(&mut inner, key);
        }

        // Insert new page into T1 MRU
        inner.t1.insert(key.clone(), ());
        self.size.fetch_add(1, Ordering::Relaxed);
    }

    fn record_removal(&self, key: &K) {
        let mut inner = self.inner.lock();
        if inner.t1.remove(key).is_some() || inner.t2.remove(key).is_some() {
            self.size.fetch_sub(1, Ordering::Relaxed);
        }
        // Also remove from ghost lists if present (optional)
        inner.b1.retain(|x| x != key);
        inner.b2.retain(|x| x != key);
    }

    fn evict(&self) -> Option<K> {
        let mut inner = self.inner.lock();
        // Evict from T1 (LRU) or T2 (LRU) depending on p
        if !inner.t1.is_empty() {
            if let Some((k, _)) = inner.t1.pop_front() {
                self.size.fetch_sub(1, Ordering::Relaxed);
                return Some(k);
            }
        }
        if !inner.t2.is_empty() {
            if let Some((k, _)) = inner.t2.pop_front() {
                self.size.fetch_sub(1, Ordering::Relaxed);
                return Some(k);
            }
        }
        None
    }

    fn clear(&self) {
        let mut inner = self.inner.lock();
        inner.t1.clear();
        inner.t2.clear();
        inner.b1.clear();
        inner.b2.clear();
        inner.p = 0;
        self.size.store(0, Ordering::Relaxed);
    }
}
```

### `src/distributed/ring.rs`

Simplified consistent hash ring with virtual nodes.

```rust
use std::collections::BTreeMap;
use std::hash::{Hash, Hasher};
use std::sync::Arc;

pub trait Node: Send + Sync + Clone {
    fn id(&self) -> String;
    fn address(&self) -> String; // e.g., "http://node1:8080"
}

pub struct ConsistentHashRing<N: Node> {
    ring: BTreeMap<u64, N>,
    virtual_nodes: usize,
}

impl<N: Node> ConsistentHashRing<N> {
    pub fn new(virtual_nodes: usize) -> Self {
        Self {
            ring: BTreeMap::new(),
            virtual_nodes,
        }
    }

    pub fn add_node(&mut self, node: N) {
        for i in 0..self.virtual_nodes {
            let key = self.hash(format!("{}:{}", node.id(), i));
            self.ring.insert(key, node.clone());
        }
    }

    pub fn remove_node(&mut self, node: &N) {
        for i in 0..self.virtual_nodes {
            let key = self.hash(format!("{}:{}", node.id(), i));
            self.ring.remove(&key);
        }
    }

    pub fn get_node(&self, key: &str) -> Option<&N> {
        if self.ring.is_empty() {
            return None;
        }
        let hash = self.hash(key);
        let (_, node) = self.ring.range(hash..).next()
            .or_else(|| self.ring.iter().next())?;
        Some(node)
    }

    fn hash(&self, input: String) -> u64 {
        let mut hasher = std::collections::hash_map::DefaultHasher::new();
        input.hash(&mut hasher);
        hasher.finish()
    }
}
```

```rust
        Self { id, addr }
    }

    async fn get_remote(&self, key: &str) -> Option<Vec<u8>> {
        let client = Client::new();
        let url = format!("{}/cache/{}", self.addr, key);
        let resp = client.get(&url).send().await.ok()?;
        if resp.status().is_success() {
            resp.bytes().await.ok().map(|b| b.to_vec())
        } else {
            None
        }
    }

    async fn set_remote(&self, key: String, value: Vec<u8>, ttl: Option<u64>) -> bool {
        let client = Client::new();
        let url = format!("{}/cache", self.addr);
        let body = SetRequest { key, value, ttl };
        let resp = client.post(&url).json(&body).send().await.ok()?;
        Some(resp.status().is_success()).unwrap_or(false)
    }
}

#[derive(Serialize, Deserialize)]
struct SetRequest {
    key: String,
    value: Vec<u8>,
    ttl: Option<u64>,
}

impl Node for HttpNode {
    fn id(&self) -> String { self.id.clone() }
    fn address(&self) -> String { self.addr.clone() }
}

pub struct DistributedCache<K, V> {
    local: Arc<Cache<K, V>>,
    ring: Arc<RwLock<ConsistentHashRing<HttpNode>>>,
    // Serialization for keys/values – for simplicity use bincode or similar.
    // Here we assume K and V are serializable.
    _marker: std::marker::PhantomData<(K, V)>,
}

pub struct DistributedConfig {
    pub virtual_nodes: usize,
    pub local_capacity: usize,
    pub local_policy: Policy,
    pub local_ttl_secs: Option<u64>,
}

impl<K, V> DistributedCache<K, V>
where
    K: Clone + Eq + Hash + Send + Sync + 'static + serde::Serialize + for<'de> serde::Deserialize<'de>,
    V: Clone + Send + Sync + 'static + serde::Serialize + for<'de> serde::Deserialize<'de>,
{
    pub fn new(config: DistributedConfig, local_cache: Option<Arc<Cache<K, V>>>) -> Self {
        let local = local_cache.unwrap_or_else(|| {
            Arc::new(
                Cache::builder()
                    .capacity(config.local_capacity)
                    .policy(config.local_policy)
                    .ttl_secs(config.local_ttl_secs.unwrap_or(0))
                    .build()
            )
        });
        Self {
            local,
            ring: Arc::new(RwLock::new(ConsistentHashRing::new(config.virtual_nodes))),
            _marker: std::marker::PhantomData,
        }
    }

    pub fn add_node(&self, id: String, addr: String) {
        let node = HttpNode::new(id, addr);
        self.ring.write().add_node(node);
    }

    pub fn remove_node(&self, id: &str) {
        // find node by id and remove
        // For simplicity, we assume we can remove by exact node.
        let mut ring = self.ring.write();
        // need to iterate to find node – omitted for brevity
    }

    pub async fn get(&self, key: K) -> Option<V> {
        let key_str = serde_json::to_string(&key).ok()?;
        // First check local
        if let Some(val) = self.local.get(&key) {
            return Some(val);
        }
        // Route to primary node
        let ring = self.ring.read();
        if let Some(node) = ring.get_node(&key_str) {
            if let Some(bytes) = node.get_remote(&key_str).await {
                if let Ok(val) = bincode::deserialize(&bytes) {
                    // Store in local for future
                    self.local.insert(key.clone(), val.clone());
                    return Some(val);
                }
            }
        }
        None
    }

    pub async fn insert(&self, key: K, value: V, ttl_secs: Option<u64>) {
        let key_str = serde_json::to_string(&key).unwrap();
        let val_bytes = bincode::serialize(&value).unwrap();
        let ring = self.ring.read();
        if let Some(node) = ring.get_node(&key_str) {
            node.set_remote(key_str, val_bytes, ttl_secs).await;
        }
        // Also store locally (write-through)
        self.local.insert_with_ttl(key, value, ttl_secs);
    }
}
```

*Note: For production, you’d implement health checks, retries, replica reads/writes, and a proper serialization scheme.*

## 3. Async API

We provide an `AsyncCache` trait and implement it for `Cache` using `tokio::task::spawn_blocking` to offload synchronous operations. This allows non‑blocking usage in async contexts.

### Add to `Cargo.toml` (already has optional tokio)

```toml
[features]

```

### `src/async_cache.rs`

```rust
use crate::Cache;
use std::future::Future;
use std::pin::Pin;
use std::hash::Hash;

/// Async version of the cache operations.
pub trait AsyncCache<K, V> {
    async fn get_async(&self, key: K) -> Option<V>;
    async fn insert_async(&self, key: K, value: V);
    async fn remove_async(&self, key: &K) -> Option<V>;
    async fn clear_async(&self);
}

impl<K, V> AsyncCache<K, V> for Cache<K, V>
where
    K: Hash + Eq + Clone + Send + Sync + 'static,
    V: Clone + Send + Sync + 'static,
{
    async fn get_async(&self, key: K) -> Option<V> {
        let cache = self.clone();
        tokio::task::spawn_blocking(move || {
            cache.get(&key)
        }).await.unwrap_or(None)
    }

    async fn insert_async(&self, key: K, value: V) {
        let cache = self.clone();
        tokio::task::spawn_blocking(move || {
            cache.insert(key, value);
        }).await.ok();
    }

    async fn remove_async(&self, key: &K) -> Option<V> {
        let cache = self.clone();
        let key = key.clone();
        tokio::task::spawn_blocking(move || {
            cache.remove(&key)
        }).await.unwrap_or(None)
    }

    async fn clear_async(&self) {
        let cache = self.clone();
        tokio::task::spawn_blocking(move || {
            cache.clear();
        }).await.ok();
    }
}
```

If you want a truly async‑native cache (e.g., using `tokio::sync::Mutex` and `async` aware eviction), that would require rewriting the entire internals. The above wrapper is sufficient for most use cases where the cache operations are fast and you just need to avoid blocking the async executor.

## 2. Persistent Disk Tier (sled)

We create a `DiskTier` struct that implements the `Tier` trait from previous code, using `sled` as the embedded database. Each node can have a disk tier as a fallback (L3) behind the in‑memory cache (L1). For simplicity, we treat it as a persistent cache with TTL.

**`src/tier/sled_tier.rs`**:

```rust
use crate::tier::Tier;
use parking_lot::RwLock;
use sled::Db;
use std::collections::HashMap;
use std::hash::Hash;
use std::sync::Arc;
use std::time::{Duration, SystemTime};

pub struct SledTier {
    db: Db,
    ttl: Option<Duration>,
}

impl SledTier {
    pub fn new(path: &str, ttl_secs: Option<u64>) -> Self {
        let db = sled::open(path).expect("Failed to open sled DB");
        Self {
            db,
            ttl: ttl_secs.map(Duration::from_secs),
        }
    }
}

impl<K, V> Tier<K, V> for SledTier
where
    K: Hash + Eq + serde::Serialize + for<'de> serde::Deserialize<'de> + Send + Sync,
    V: serde::Serialize + for<'de> serde::Deserialize<'de> + Clone + Send + Sync,
{
    fn get(&self, key: &K) -> Option<V> {
        let key_bytes = bincode::serialize(key).ok()?;
        let value_bytes = self.db.get(&key_bytes).ok()??;
        // Optionally check TTL stored in metadata
        let (value, expiry): (V, Option<u64>) = bincode::deserialize(&value_bytes).ok()?;
        if let Some(exp) = expiry {
            let now = SystemTime::now()
                .duration_since(SystemTime::UNIX_EPOCH)
                .unwrap()
                .as_secs();
            if now >= exp {
                let _ = self.db.remove(&key_bytes);
                return None;
            }
        }
        Some(value)
    }

    fn set(&self, key: K, value: V) {
        let expiry = self.ttl.map(|dur| {
            SystemTime::now()
                .duration_since(SystemTime::UNIX_EPOCH)
                .unwrap()
                .as_secs()
                + dur.as_secs()
        });
        let to_store = (value, expiry);
        let key_bytes = bincode::serialize(&key).unwrap();
        let val_bytes = bincode::serialize(&to_store).unwrap();
        let _ = self.db.insert(key_bytes, val_bytes);
    }

    fn del(&self, key: &K) {
        let key_bytes = bincode::serialize(key).unwrap();
        let _ = self.db.remove(key_bytes);
    }
}
```

## 3. Failure Detection and Ring Rebalancing

We implement a **gossip‑style failure detector** (simplified: heartbeat + configurable timeout). Each node periodically sends heartbeats to a subset of other nodes. If a node fails to respond for a threshold, it is marked dead and removed from the consistent hash ring. When it recovers, it is re‑added.

Because the ring is global, we need a **cluster membership manager** that runs on each node and communicates via gRPC. The design:

- Each node knows the initial seed nodes.
- Nodes periodically exchange membership lists (using a gossip protocol).
- When a node is detected as dead (no heartbeat for T seconds), the local ring is updated (node removed).
- When a dead node becomes reachable again, it rejoins (re‑added to ring).

For simplicity, we implement a **centralized coordinator** (or one node acting as leader) that maintains the ring and pushes updates. But for true distribution, we use a **gossip-based membership** (e.g., SWIM). The code below provides a basic version using a leader‑election approach with a single “control” node – but for production a full SWIM is recommended.

We will provide the core structures for a **distributed failure detector** that can be integrated.

```rust
use std::collections::HashMap;
use std::sync::Arc;
use parking_lot::RwLock;
use tokio::sync::broadcast;
use serde::{Serialize, Deserialize};

#[derive(Clone, Debug, Serialize, Deserialize, PartialEq, Eq)]
pub enum NodeState {
    Alive,
    Suspect,
    Dead,
}

#[derive(Clone, Debug, Serialize, Deserialize)]
pub struct NodeInfo {
    pub id: String,
    pub addr: String,   // gRPC address
    pub state: NodeState,
    pub last_heartbeat: u64, // unix timestamp
}

pub struct ClusterMembership {
    nodes: Arc<RwLock<HashMap<String, NodeInfo>>>,
    updates_tx: broadcast::Sender<ClusterEvent>,
}

#[derive(Clone, Debug)]
pub struct ClusterEvent {
    pub node_id: String,
    pub new_state: NodeState,
}

impl ClusterMembership {
    pub fn new() -> Self {
        let (tx, _) = broadcast::channel(100);
        Self {
            nodes: Arc::new(RwLock::new(HashMap::new())),
            updates_tx: tx,
        }
    }

    pub fn add_node(&self, node: NodeInfo) {
        let mut nodes = self.nodes.write();
        if let Some(existing) = nodes.get(&node.id) {
            if existing.state == node.state && existing.addr == node.addr {
                return;
            }
        }
        nodes.insert(node.id.clone(), node);
        let _ = self.updates_tx.send(ClusterEvent {
            node_id: node.id,
            new_state: node.state,
        });
    }

    pub fn update_state(&self, node_id: &str, state: NodeState) {
        let mut nodes = self.nodes.write();
        if let Some(mut node) = nodes.get_mut(node_id) {
            node.state = state;
            let _ = self.updates_tx.send(ClusterEvent {
                node_id: node_id.to_string(),
                new_state: state,
            });
        }
    }

    pub fn get_alive_nodes(&self) -> Vec<NodeInfo> {
        self.nodes
            .read()
            .values()
            .filter(|n| n.state == NodeState::Alive)
            .cloned()
            .collect()
    }

    pub fn subscribe(&self) -> broadcast::Receiver<ClusterEvent> {
        self.updates_tx.subscribe()
    }
}
```

```rust
use crate::cache::cache_service_server::{CacheService, CacheServiceServer};
use crate::cache::{GetRequest, GetResponse, SetRequest, SetResponse, DeleteRequest, DeleteResponse};
use crate::Cache; // our local cache (with optional disk tier)
use std::sync::Arc;

pub struct CacheGrpcServer<K, V> {
    cache: Arc<Cache<K, V>>,
    _marker: std::marker::PhantomData<(K, V)>,
}

impl<K, V> CacheGrpcServer<K, V> {
    pub fn new(cache: Arc<Cache<K, V>>) -> Self {
        Self { cache, _marker: std::marker::PhantomData }
    }
}

#[tonic::async_trait]
impl<K, V> CacheService for CacheGrpcServer<K, V>
where
    K: serde::Serialize + for<'de> serde::Deserialize<'de> + Send + Sync + 'static,
    V: serde::Serialize + for<'de> serde::Deserialize<'de> + Clone + Send + Sync + 'static,
{
    async fn get(&self, req: Request<GetRequest>) -> Result<Response<GetResponse>, Status> {
        let key_bytes = req.into_inner().key;
        let key: K = bincode::deserialize(&key_bytes).map_err(|_| Status::invalid_argument("invalid key"))?;
        let value = self.cache.get(&key);
        if let Some(val) = value {
            let val_bytes = bincode::serialize(&val).unwrap();
            Ok(Response::new(GetResponse { value: val_bytes, found: true }))
        } else {
            Ok(Response::new(GetResponse { value: vec![], found: false }))
        }
    }

    async fn set(&self, req: Request<SetRequest>) -> Result<Response<SetResponse>, Status> {
        let req = req.into_inner();
        let key: K = bincode::deserialize(&req.key).map_err(|_| Status::invalid_argument("invalid key"))?;
        let value: V = bincode::deserialize(&req.value).map_err(|_| Status::invalid_argument("invalid value"))?;
        self.cache.insert_with_ttl(key, value, if req.ttl_secs > 0 { Some(req.ttl_secs) } else { None });
        Ok(Response::new(SetResponse { success: true }))
    }

    async fn delete(&self, req: Request<DeleteRequest>) -> Result<Response<DeleteResponse>, Status> {
        let key: K = bincode::deserialize(&req.into_inner().key).map_err(|_| Status::invalid_argument("invalid key"))?;
        self.cache.remove(&key);
        Ok(Response::new(DeleteResponse { success: true }))
    }
}
```

## 5. Replication and Ring Rebalancing

We enhance the `DistributedCache` to:

- Write to **N** replicas (N = replication factor). Uses the consistent hash ring to get the primary node plus N‑1 successors.
- Read from **any** node that has the key (attempt primary, then replicas, or broadcast).
- On node failure (detected via membership changes), rebuild the local ring and re‑replicate data (full redistribution is complex; we implement a lazy approach – on write, data is written to current replicas; on read, we may fetch from a node that still holds it if replica set changed; full rebalancing is omitted for brevity but can be added with background migration).

**`src/distributed/replication.rs`** (new module):

```rust
use super::ring::{ConsistentHashRing, Node};
use crate::grpc::client::CacheGrpcClient;
use std::sync::Arc;
use parking_lot::RwLock;
use tokio::sync::Semaphore;

pub struct ReplicatedCache<N: Node> {
    ring: Arc<RwLock<ConsistentHashRing<N>>>,
    replication_factor: usize,
    // Node clients cache (map addr -> client)
    clients: Arc<RwLock<HashMap<String, CacheGrpcClient>>>,
    concurrency_limiter: Arc<Semaphore>,
}

impl<N: Node> ReplicatedCache<N> {
    pub fn new(ring: Arc<RwLock<ConsistentHashRing<N>>>, replication_factor: usize) -> Self {
        Self {
            ring,
            replication_factor,
            clients: Arc::new(RwLock::new(HashMap::new())),
            concurrency_limiter: Arc::new(Semaphore::new(100)),
        }
    }

    async fn get_clients(&self, addrs: Vec<String>) -> Vec<CacheGrpcClient> {
        let mut out = Vec::new();
        let mut clients_guard = self.clients.write();
        for addr in addrs {
            let client = clients_guard.entry(addr.clone())
                .or_insert_with(|| {
                    // This blocks in async context – better to use async init, but simplified.
                    CacheGrpcClient::connect(addr).await.unwrap()
                })
                .clone();
            out.push(client);
        }
        out
    }

    pub async fn get<K, V>(&self, key: K) -> Option<V>
    where
        K: serde::Serialize + Clone + Send + Sync,
        V: for<'de> serde::Deserialize<'de> + Clone + Send,
    {
        let key_str = serde_json::to_string(&key).unwrap();
        let ring = self.ring.read();
        let primary = ring.get_node(&key_str).cloned();
        if let Some(node) = primary {
            let addr = node.address();
            let mut client = self.get_clients(vec![addr]).await.into_iter().next().unwrap();
            if let Ok(val) = client.get(key.clone()).await {
                return Some(val);
            }
            // else fallback to next replicas? For simplicity, return None.
        }
        None
    }

    pub async fn set<K, V>(&self, key: K, value: V, ttl_secs: Option<u64>)
    where
        K: serde::Serialize + Clone + Send + Sync,
        V: serde::Serialize + Clone + Send,
    {
        let key_str = serde_json::to_string(&key).unwrap();
        let ring = self.ring.read();
        let mut replicas = ring.get_replicas(&key_str, self.replication_factor); // custom method on ring
        let addrs: Vec<String> = replicas.iter().map(|n| n.address()).collect();
        let clients = self.get_clients(addrs).await;
        let ttl = ttl_secs.unwrap_or(0);
        let mut tasks = Vec::new();
        for mut client in clients {
            let key = key.clone();
            let value = value.clone();
            tasks.push(tokio::spawn(async move {
                let _ = client.set(key, value, ttl).await;
            }));
        }
        for task in tasks {
            let _ = task.await;
        }
    }
```

```rust
impl<N: Node> ConsistentHashRing<N> {
    pub fn get_replicas(&self, key: &str, count: usize) -> Vec<N> {
        let hash = self.hash(key.to_string());
        let mut results = Vec::new();
        let mut iter = self.ring.range(hash..);
        while results.len() < count {
            if let Some((_, node)) = iter.next() {
                if !results.iter().any(|n: &N| n.id() == node.id()) {
                    results.push(node.clone());
                }
            } else {
                iter = self.ring.iter();
            }
        }
        results
    }
}
```

## 1. Full Failure Detector (SWIM‑based)

The detector runs on each node. It periodically pings a random subset of alive nodes, uses indirect pings if a node becomes suspect, and updates the cluster membership state.

### `src/cluster/failure_detector.rs` (complete)

```rust
use super::membership::{ClusterMembership, NodeState, NodeInfo};
use crate::grpc::client::CacheGrpcClient;
use tokio::time::{self, Duration};
use tokio::sync::RwLock as TokioRwLock;
use std::collections::HashSet;
use std::sync::Arc;
use rand::seq::SliceRandom;
use log::{info, warn, error};

pub struct FailureDetector {
    membership: Arc<ClusterMembership>,
    local_id: String,
    ping_interval: Duration,
    suspect_timeout: Duration,
    dead_timeout: Duration,
    // For gossip: we maintain a list of known nodes and their last contact.
    // We also need a way to send "ping-req" to other nodes.
}

impl FailureDetector {
    pub fn new(
        membership: Arc<ClusterMembership>,
        local_id: String,
        ping_interval: Duration,
        suspect_timeout: Duration,
        dead_timeout: Duration,
    ) -> Self {
        Self {
            membership,
            local_id,
            ping_interval,
            suspect_timeout,
            dead_timeout,
        }
    }

    /// Start the failure detection loop.
    /// `grpc_client_factory` creates a client for a given node address.
    pub async fn run<F, Fut>(&self, grpc_client_factory: F)
    where
        F: Fn(String) -> Fut + Send + Sync,
        Fut: std::future::Future<Output = CacheGrpcClient> + Send,
    {
        let mut ping_ticker = time::interval(self.ping_interval);
        let mut suspect_check_ticker = time::interval(self.suspect_timeout / 2);
        let local_id = self.local_id.clone();

        loop {
            tokio::select! {
                _ = ping_ticker.tick() => {
                    let alive = self.membership.get_alive_nodes();
                    // Exclude self
                    let targets: Vec<NodeInfo> = alive.into_iter()
                        .filter(|n| n.id != local_id)
                        .collect();
                    if let Some(target) = targets.choose(&mut rand::thread_rng()) {
                        let client = grpc_client_factory(target.addr.clone()).await;
                        self.ping_node(target, client).await;
                    }
                }
                _ = suspect_check_ticker.tick() => {
                    self.update_suspect_and_dead_nodes().await;
                }
            }
        }
    }

    async fn ping_node(&self, node: &NodeInfo, mut client: CacheGrpcClient) {
        // Ping via gRPC (we need a lightweight Ping method; if not present, use Get on a dummy key).
        let start = std::time::Instant::now();
        let result = tokio::time::timeout(self.suspect_timeout / 2, async {
            client.ping().await // we must add a Ping method to proto
        }).await;

        match result {
            Ok(Ok(_)) => {
                // Node alive
                self.membership.update_heartbeat(&node.id).await;
                if self.membership.get_state(&node.id).await == NodeState::Suspect {
                    self.membership.update_state(&node.id, NodeState::Alive).await;
                    info!("Node {} recovered from suspect", node.id);
                }
            }
            Ok(Err(e)) => {
                // gRPC error – treat as suspicious
                warn!("Ping to {} failed: {}", node.id, e);
                self.maybe_mark_suspect(&node.id).await;
            }
            Err(_timeout) => {
                warn!("Ping to {} timed out", node.id);
                self.maybe_mark_suspect(&node.id).await;
            }
        }
    }

    async fn maybe_mark_suspect(&self, node_id: &str) {
        let state = self.membership.get_state(node_id).await;
        if state == NodeState::Alive {
            self.membership.update_state(node_id, NodeState::Suspect).await;
            info!("Marked {} as suspect", node_id);
            // Schedule indirect ping (ping-req) via a third node.
            self.request_indirect_ping(node_id).await;
        }
    }

    async fn request_indirect_ping(&self, suspect_id: &str) {
        // Pick a random alive node (not self, not suspect)
        let alive = self.membership.get_alive_nodes();
        let helper = alive.iter()
            .find(|n| n.id != self.local_id && n.id != suspect_id)
            .cloned();
        if let Some(helper) = helper {
            // Send a "ping-req" gRPC call to helper asking it to ping suspect.
            // We need to add a PingReq method in proto.
            // Implementation omitted for brevity – similar to ping but with a target.
        }
    }

    async fn update_suspect_and_dead_nodes(&self) {
        let now = std::time::SystemTime::now()
            .duration_since(std::time::UNIX_EPOCH)
            .unwrap()
            .as_secs();
        let all_nodes = self.membership.get_all_nodes().await;
        for node in all_nodes {
            if node.id == self.local_id { continue; }
            let last_contact = self.membership.get_last_heartbeat(&node.id).await;
            let elapsed = now.saturating_sub(last_contact);
            match node.state {
                NodeState::Suspect if elapsed >= self.dead_timeout.as_secs() => {
                    self.membership.update_state(&node.id, NodeState::Dead).await;
                    error!("Node {} marked dead", node.id);
                    // Trigger ring rebuild and re-replication (via event broadcast)
                    let _ = self.membership.trigger_ring_rebuild().await;
                }
                NodeState::Alive if elapsed >= self.suspect_timeout.as_secs() => {
                    self.maybe_mark_suspect(&node.id).await;
                }
                _ => {}
            }
        }
    }
}
```

## 2. Background Re‑replication

When the membership changes (node added/removed), the consistent hash ring changes. Keys that previously belonged to a dead node (or new node) must be re‑replicated. We implement a background task that:

- Listens to `ClusterEvent` from `ClusterMembership`.
- When a node is marked dead or a new node joins, it triggers a full re‑replication scan. Because scanning all keys is expensive in a large cache, we can use **lazy re‑replication** (on access, fix missing replicas) + a background batch job that re‑hashes a sample of keys.

We'll implement **full re‑replication** for simplicity, but note that in production you would use incremental approaches (e.g., consistent hashing with token ranges).

### `src/distributed/re_replication.rs`

```rust
use crate::cluster::{ClusterMembership, NodeState};
use crate::distributed::ring::ConsistentHashRing;
use crate::distributed::ReplicatedCache;
use crate::grpc::client::CacheGrpcClient;
use crate::Cache; // local cache
use std::collections::HashSet;
use std::sync::Arc;
use tokio::sync::RwLock;
use tokio_stream::wrappers::BroadcastStream;
use futures::StreamExt;
use log::info;

pub struct ReReplicationManager<K, V> {
    membership: Arc<ClusterMembership>,
    ring: Arc<RwLock<ConsistentHashRing<HttpNode>>>, // your node type
    local_cache: Arc<Cache<K, V>>,
    replication_factor: usize,
    // Use a client pool to talk to other nodes
}

impl<K, V> ReReplicationManager<K, V>
where
    K: Clone + Eq + Hash + Send + Sync + 'static + serde::Serialize + for<'de> serde::Deserialize<'de>,
    V: Clone + Send + Sync + 'static + serde::Serialize + for<'de> serde::Deserialize<'de>,
{
    pub fn new(
        membership: Arc<ClusterMembership>,
        ring: Arc<RwLock<ConsistentHashRing<HttpNode>>>,
        local_cache: Arc<Cache<K, V>>,
        replication_factor: usize,
    ) -> Self {
        Self { membership, ring, local_cache, replication_factor }
    }

    pub async fn start(&self) {
        let mut event_stream = BroadcastStream::new(self.membership.subscribe());
        while let Some(event) = event_stream.next().await {
            if let Ok(ev) = event {
                if matches!(ev.new_state, NodeState::Dead | NodeState::Alive) {
                    info!("Cluster change: {:?} -> {}. Starting re-replication", ev.node_id, ev.new_state);
                    self.re_replicate().await;
                }
            }
        }
    }

    async fn re_replicate(&self) {
        // Step 1: Build the current ring (primary owners for all keys)
        let ring = self.ring.read().await;
        // Step 2: Iterate over all keys in local cache (expensive!)
        // For production, you'd store a set of keys or use a cursor.
        // We assume local_cache provides a `iter_keys()` method.
        let keys: Vec<K> = self.local_cache.iter_keys().await; // custom method
        for key in keys {
            let key_str = serde_json::to_string(&key).unwrap();
            let expected_owners = ring.get_replicas(&key_str, self.replication_factor);
            let expected_ids: HashSet<String> = expected_owners.iter().map(|n| n.id()).collect();

            // Find which nodes currently have this key (query replicas)
            // This is expensive – we can instead rely on eventual consistency:
            // For each missing node, we push the key to it.
            for node in expected_owners {
                let node_id = node.id();
                // Skip self – local cache already has it
                if node_id == self.membership.local_id() { continue; }
                // Check if the remote node has the key (optional)
                // To avoid network flood, we can assume missing and send.
                self.replicate_to_node(&key, &node).await;
            }
        }
        info!("Re-replication finished");
    }

    async fn replicate_to_node(&self, key: &K, node: &HttpNode) {
        if let Some(value) = self.local_cache.get(key) {
            let mut client = CacheGrpcClient::connect(node.address()).await.unwrap();
            let _ = client.set(key.clone(), value, None).await;
        }
    }
}
```

## 2. Efficient Incremental Re‑replication

The naive approach scans all keys on every membership change. With millions of keys, this is prohibitive.

### 2.1. Core Idea: Hash‑Range Partitioning + Merkle Tree

- Each node stores a **consistent hash ring** and knows which hash ranges it owns.
- The local cache maintains a **B‑tree index mapping key hash → key** (using `sled` for persistence, or an in‑memory `std::collections::BTreeMap`).
- On membership change, the ring changes. For each key range that moves from node A to node B:
    - Node A sends to Node B a list of hashes (or a Merkle tree root) for that range.
    - Node B compares and requests missing keys.
- This reduces network and CPU overhead drastically.

We implement a simplified version: **range‑based syncing** where each node exports a list of key hashes in a given hash range (via gRPC). The re‑replication manager then compares hash sets between nodes.

```rust
pub struct HashIndex {
    db: Db,

impl HashIndex {
    pub fn new(path: &str) -> Self {
        let db = sled::open(path).expect("Failed to open hash index DB");
        Self { db }
    }

    pub fn insert<K: serde::Serialize>(&self, key: &K, hash: u64) {
        let key_bytes = bincode::serialize(key).unwrap();
        let _ = self.db.insert(hash.to_be_bytes(), key_bytes);
    }

    pub fn remove<K: serde::Serialize>(&self, key: &K) {
        // Need to know hash to remove; caller provides
    }

    pub fn get_keys_in_range(&self, start: u64, end: u64) -> Vec<u64> {
        let mut hashes = Vec::new();
        for res in self.db.range(start.to_be_bytes()..=end.to_be_bytes()) {
            if let Ok((hash_bytes, _)) = res {
                let hash = u64::from_be_bytes(hash_bytes.as_ref().try_into().unwrap());
                hashes.push(hash);
            }
        }
        hashes
    }

    pub fn get_key_by_hash(&self, hash: u64) -> Option<Vec<u8>> {
        self.db.get(hash.to_be_bytes()).ok().flatten().map(|v| v.to_vec())
    }
}
```

### 2.5. Incremental Re‑replication Algorithm

Now the `ReReplicationManager` can compute which hash ranges moved due to a ring change. We use a **range comparison** strategy:

1. **Compute old and new ring** (before and after membership change).
2. **For each node `X`**, compute the hash ranges that `X` **lost** (i.e., old ownership but not new).
3. **For each lost range `[start, end]`**, the old owner (which may now be dead) can be used if still reachable; otherwise, we rely on replicas.
4. For each such range, the current owners (new replicas) fetch the list of hashes from the old owner (if alive) or from any node that still holds the data.
5. Then they request missing keys using `GetKeyByHash`.

Simpler approach used in production (e.g., in DynamoDB): each node periodically exchanges a **Merkle tree** of its key hashes with other nodes. We'll implement a basic Merkle tree for a given hash range.

**`src/merkle.rs`**:

```rust
use sha2::{Sha256, Digest};
use std::collections::BTreeMap;

pub struct MerkleTree {
    root: Vec<u8>,
    leaves: Vec<Vec<u8>>,
}

impl MerkleTree {
    pub fn from_hashes(hashes: &[u64]) -> Self {
        let leaves: Vec<Vec<u8>> = hashes.iter().map(|h| {
            let mut hasher = Sha256::new();
            hasher.update(h.to_be_bytes());
            hasher.finalize().to_vec()
        }).collect();
        let root = Self::build_root(&leaves);
        Self { root, leaves }
    }

    fn build_root(leaves: &[Vec<u8>]) -> Vec<u8> {
        if leaves.is_empty() {
            return vec![0u8; 32];
        }
        let mut level = leaves.to_vec();
        while level.len() > 1 {
            let mut next = Vec::new();
            for chunk in level.chunks(2) {
                let mut hasher = Sha256::new();
                hasher.update(&chunk[0]);
                if chunk.len() > 1 {
                    hasher.update(&chunk[1]);
                } else {
                    hasher.update(&chunk[0]);
                }
                next.push(hasher.finalize().to_vec());
            }
            level = next;
        }
        level[0].clone()
    }

    pub fn root(&self) -> &[u8] { &self.root }
}
```

### `src/distributed/incremental_re_replication.rs`

```rust
use crate::cluster::{ClusterMembership, NodeState};
use crate::distributed::ring::ConsistentHashRing;
use crate::grpc::client::CacheGrpcClient;
use crate::Cache;
use std::collections::{HashSet, BTreeSet};
use std::sync::Arc;
use tokio::sync::RwLock;
use tokio_stream::wrappers::BroadcastStream;
use futures::StreamExt;
use log::info;

pub struct IncrementalReReplication<K, V> {
    membership: Arc<ClusterMembership>,
    ring: Arc<RwLock<ConsistentHashRing<HttpNode>>>,
    local_cache: Arc<Cache<K, V>>,
    replication_factor: usize,
    // Caches the last known ring version
    last_ring: Arc<RwLock<Option<ConsistentHashRing<HttpNode>>>>,
}

impl<K, V> IncrementalReReplication<K, V>
where
    K: Clone + Eq + Hash + Send + Sync + 'static + serde::Serialize + for<'de> serde::Deserialize<'de>,
    V: Clone + Send + Sync + 'static + serde::Serialize + for<'de> serde::Deserialize<'de>,
{
    pub fn new(
        membership: Arc<ClusterMembership>,
        ring: Arc<RwLock<ConsistentHashRing<HttpNode>>>,
        local_cache: Arc<Cache<K, V>>,
        replication_factor: usize,
    ) -> Self {
        Self {
            membership,
            ring,
            local_cache,
            replication_factor,
            last_ring: Arc::new(RwLock::new(None)),
        }
    }

    pub async fn start(&self) {
        let mut event_stream = BroadcastStream::new(self.membership.subscribe());
        while let Some(event) = event_stream.next().await {
            if let Ok(ev) = event {
                if matches!(ev.new_state, NodeState::Dead | NodeState::Alive) {
                    info!("Cluster change detected, syncing hash ranges");
                    self.sync_ranges().await;
                }
            }
        }
    }

    async fn sync_ranges(&self) {
        // Take current ring
        let new_ring = self.ring.read().await.clone();
        let old_ring_opt = self.last_ring.write().await.take();

        let old_ring = match old_ring_opt {
            Some(old) => old,
            None => {
                // First run, store ring and return (no sync needed)
                *self.last_ring.write().await = Some(new_ring);
                return;
            }
        };

        // Compute which hash ranges this node lost and gained
        let my_id = self.membership.local_id();
        let old_my_ranges = old_ring.get_ownership_ranges(&my_id);
        let new_my_ranges = new_ring.get_ownership_ranges(&my_id);

        // Ranges I lost: in old but not in new
        let lost_ranges = old_my_ranges.difference(&new_my_ranges);
        for range in lost_ranges {
            self.handle_lost_range(range).await;
        }

        // Ranges I gained: in new but not in old
        let gained_ranges = new_my_ranges.difference(&old_my_ranges);
        for range in gained_ranges {
            self.handle_gained_range(range).await;
        }

        *self.last_ring.write().await = Some(new_ring);
    }

    async fn handle_lost_range(&self, range: &(u64, u64)) {
        // For each key in this range, ensure it is still replicated on other nodes.
        // We can delegate to other nodes' background sync; no immediate action needed.
        info!("Lost range [{:x}, {:x}], keys will be picked up by new owners", range.0, range.1);
    }

    async fn handle_gained_range(&self, range: &(u64, u64)) {
        // We need to fetch keys in this range from existing replicas.
        // Find the old owner(s) (nodes that were responsible for this range in old ring)
        let old_ring = self.last_ring.read().await.as_ref().unwrap();
        let old_owner_ids = old_ring.get_nodes_for_range(range.0, range.1);
        for node_id in old_owner_ids {
            if let Some(node) = self.membership.get_node(&node_id).await {
                if node.state == NodeState::Alive {
                    self.fetch_range_from_node(range, &node).await;
                    break; // only need one source
                }
            }
        }
    }

    async fn fetch_range_from_node(&self, range: &(u64, u64), node: &NodeInfo) {
        let mut client = CacheGrpcClient::connect(node.addr.clone()).await.unwrap();
        let response = client.get_keys_in_range(range.0, range.1).await.unwrap();
        let hashes = response.hashes;
        for hash in hashes {
            let key_bytes = client.get_key_by_hash(hash).await.unwrap().key;
            let key: K = bincode::deserialize(&key_bytes).unwrap();
            // Fetch the actual value from the remote node (or maybe we have it locally)
            // For simplicity, we fetch value via normal get.
            if let Some(value) = client.get(key.clone()).await {
                self.local_cache.insert(key, value);
            }
        }
    }
}
```

We must add the helper methods `get_ownership_ranges`, `get_nodes_for_range` to `ConsistentHashRing`. These use the ring's `BTreeMap` to compute which hash intervals belong to which node. Implementation is straightforward using the ordering of nodes on the ring.

## 🔍 Critical Audit of the Original Plan

### 1. “Implementing the WireGuard protocol as a kernel module”
**Flaw**  
WireGuard is already a mainline Linux kernel module (`wireguard.ko`) since 2020, and its userspace implementations (e.g., Cloudflare’s `boringtun`, `wireguard-go`) are highly performant. Re‑implementing it from scratch in Rust as a *kernel module* would be both redundant and a monumental engineering effort, offering no practical benefit while introducing massive maintenance and security risk. Moreover, writing kernel modules in Rust is nascent (requires `rustc` support, complex build‑system integration, and kernel‑specific crates) and is not recommended for a production system without a very strong reason.

**What should be done instead**  
Leverage the **existing, audited** WireGuard implementations. Use the kernel module when available (via netlink) for maximum performance, and fall back to a high‑performance userspace implementation (e.g., `boringtun` or a custom Rust userspace stack based on the `wireguard-rs` crate) for platforms without kernel support, in containers, and for cryptographic agility. This avoids reinventing the wheel while keeping the data path flexible.

### 2. “eBPF‑based packet‑forwarding plane” for the VPN
**Flaw**  
eBPF is excellent for fast packet filtering, redirection, and observability, but it is **not** a replacement for the WireGuard data path. WireGuard already encrypts/decrypts in the kernel (or in userspace with a TUN device). Attempting to forward VPN traffic entirely in eBPF would require re‑implementing the entire encryption/decryption and tunnel logic in eBPF bytecode, which is heavily constrained (verifier limits, no loops, no cryptographic primitives). This is not feasible for a production VPN.

**Correction**  
Use eBPF for **traffic steering and policy enforcement** before encryption/after decryption. For example, eBPF classifiers (TC) or XDP programs can:
- Redirect specific flows to the proxy or the VPN tunnel.
- Apply quality‑of‑service (QoS) rules.
- Record flow statistics without kernel‑bypass overhead.

The heavy crypto and tunnel framing remain in the WireGuard implementation.

### 3. Decentralised Control Plane via DHT and ICE
**Flaw**  
A decentralised, DHT‑based peer discovery with NAT‑piercing (ICE/STUN/TURN) is an ambitious feature but adds enormous complexity for enterprise use cases. Most enterprises deploy VPNs with a **known set of gateways** and do not require fully decentralised peer‑to‑peer connectivity across all nodes. Building a robust DHT that resists Sybil attacks, handles churn, and provides low‑latency lookups is a 6‑12 month project by itself, and combining it with ICE introduces additional failure modes. The timeline (6 months) is unrealistic with a team of 5‑7.

**Suggestion**  
Provide a **hybrid control plane**: a centralised (or clustered) coordination server for initial peer registration and policy management, with an optional **decentralised extension** for advanced use cases (e.g., mesh VPNs for distributed edge devices). This aligns with enterprise expectations while still leaving the door open for future P2P capabilities.

### 4. “Censorship evasion” with QUIC/WebSocket/gRPC imitation
**Flaw**  
Wrapping VPN traffic inside another protocol (QUIC, WebSocket, etc.) is a valid circumvention technique, but the plan mentions “eBPF TC filters or userspace traffic transformers.” Doing this with eBPF is extremely challenging because eBPF programs cannot efficiently buffer and transform streams to mimic complex protocols like QUIC; that would require a full userspace proxy. The plan conflates the roles of eBPF and userspace.

**Correction**  
Implement pluggable transport layers in **userspace**, as a separate shim that sits between the application and the WireGuard interface. This is the architecture used by tools like Outline and V2Ray. eBPF is only used for transparent redirection (e.g., catch traffic from specific apps and route it to the obfuscator).

### 5. Timeline and Team Size
**Original claim**: 24 months, 5‑7 engineers for a fully featured VPN + proxy with post‑quantum crypto, zero‑trust, eBPF, DHT, ICE, multi‑hop, SIEM integration, etc.

**Reality check**  
Even with strong open‑source leverage, building this system from scratch would require a team of **10‑12 experienced engineers** working for **30‑36 months** to achieve production‑grade reliability. The original timeline would lead to a prototype at best, missing most enterprise‑hardening.

### 3. Detailed Component Design

#### 3.1 WireGuard Integration & Post‑Quantum Security
- **Hybrid key exchange**: Use the **ROSENPASS** approach (a post‑quantum handshake daemon) that wraps WireGuard’s initial handshake, negotiating a classical (Curve25519) and a post‑quantum (e.g., ML‑KEM‑768) shared secret. The combined secret is used as the WireGuard pre‑shared key.
- **Data plane encryption**: Unchanged – ChaCha20‑Poly1305 at 256‑bit (already quantum‑resistant for the symmetric part).
- **Implementation**: The agent communicates with ROSENPASS over a local Unix socket; ROSENPASS pushes refreshed keys to the WireGuard interface periodically.

#### 3.2 eBPF Integration (Strictly for Traffic Steering & Observability)
- **XDP programs** attached to the physical NIC for extremely fast flow classification and direct kernel‑bypass forwarding to the WireGuard TUN device.
- **TC classifiers** for egress traffic: redirect traffic destined to specific CIDRs to the proxy, or into the tunnel, based on configurable rules.
- **BPF maps** shared with userspace to provide real‑time per‑flow statistics (packet/byte counts, RTT) without copying packets.

#### 3.3 Proxy Engine
- **Rust async HTTP proxy** built on `hyper`, supporting CONNECT, forward/reverse proxy, transparent proxy (via REDIRECT + eBPF).
- **SOCKS5 support** via a separate listener.
- **Load balancing**: round‑robin, least connections, and latency‑weighted across multiple upstream VPN exit nodes or external proxies.
- **Health checks** and automatic failover.

#### 3.4 Pluggable Transport Obfuscation (Censorship Circumvention)
- **Userspace obfuscation layer**: A shim that sits between the proxy/VPN and the network, applying transforms such as:
  - **Obfs4** (Tor‑style padding + scrambling).
  - **meek** (domain fronting via CDNs).
  - **WebSocket/QUIC encapsulation** with randomised payloads and timing.
- This layer is **completely decoupled** from the crypto tunnel, enabled per‑outbound route.

#### 3.5 Zero‑Trust Access Module
- **Device posture check** via integration with a custom agent (or MDM) that reports OS version, patch level, disk encryption, etc. before granting access.
- **Short‑lived mTLS certificates** issued by the internal CA after successful authentication via OIDC/SAML.
- **Continuous verification**: Every data session is bound to the client’s identity; revocation propagates in seconds via CRL or OCSP stapling.

#### 3.6 Control Plane
- **Central coordinator**: A set of stateless, horizontally‑scalable Rust services using `gRPC` and backed by etcd or a PostgreSQL database for persistence. The coordinator:
  - Registers peers and their public keys.
  - Pushes configuration (allowed IPs, routes, split‑tunnel rules) to agents.
  - Provides a REST/gRPC API for the management UI.
- **Optional decentralised mesh**: For edge‑to‑edge connections, a lightweight DHT (e.g., Kademlia) can be introduced later to distribute peer information without a central server; but this is a Phase 4 feature, not a foundation.

#### 3.7 Enterprise Features
- **RBAC** with integration to external IdPs (LDAP, Azure AD, Okta).
- **Audit logs** shipped to SIEM/SOAR via syslog, Kafka, or OpenTelemetry.
- **Multi‑tenancy**: One deployment can serve multiple isolated organisations with separate policy domains.
- **High availability**: All control plane components are stateless and can run behind a load balancer; agents cache last‑known configuration to survive control‑plane outage.
