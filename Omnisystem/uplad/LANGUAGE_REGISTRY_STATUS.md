## Integration Points

### With Frontend Loader
Each language spec feeds into `frontend_loader.ti`:
1. Load spec from registry ✅
2. Generate Omni-IR parser ✅ (placeholder, full in Wave 3)
3. Compile IR to native code ✅ (BACE integration, placeholder)
4. Cache compiled frontend ✅
5. Support hot-reload ✅

### With Hot-Reload System
All language specs are compatible with atomic hot-reloading:
- Versioned symbol tables track each language's functions
- Type descriptors enable safe data migration
- Cross-language calls routed through Omni-IR
- Zero-downtime updates for any registered language

### With Bonsai Ecosystem
Language registry integrates with:
- Model execution (Octopus, Poe, Omni-AI models)
- BonsAI SDK compilation targets
- bonsai-bot service hot-reload
- Tauri app frontend/logic reload
