2. CORE API ENDPOINTS
================================================================================

1. GET /services
   - Handler: list_services()
   - Response: ServiceListResponse { services: Vec<ServiceSummary>, total_count }
   - Purpose: List all registered services with status

2. POST /services/{name}/start
   - Handler: start_service()
   - Request: StartServiceRequest { config?, wait_for_ready? }
   - Response: StartServiceResponse { state, pid, message }
   - Purpose: Start a service with optional configuration

3. POST /services/{name}/stop
   - Handler: stop_service()
   - Request: StopServiceRequest { graceful?, timeout_seconds? }
   - Response: StopServiceResponse { state, uptime_seconds, message }
   - Purpose: Stop service gracefully or forcefully

4. GET /services/{name}/status
   - Handler: get_service_status()
   - Response: ServiceDetailResponse { state, status, cpu_percent, memory_mb, ... }
   - Purpose: Get detailed service status and metrics

5. POST /services/{name}/restart
   - Handler: restart_service()
   - Request: RestartServiceRequest { graceful? }
   - Response: RestartServiceResponse { old_pid, new_pid, message }
   - Purpose: Restart service with PID tracking

6. POST /services/{name}/configure
   - Handler: configure_service()
   - Request: ConfigureServiceRequest { config, merge? }
   - Response: ConfigureServiceResponse { config, applied, message }
   - Purpose: Apply or merge service configuration

7. POST /services/{name}/snapshot
   - Handler: snapshot_service()
   - Request: SnapshotServiceRequest { snapshot_name?, description? }
   - Response: SnapshotResponse { snapshot_id, size_bytes, timestamp }
   - Purpose: Create service state snapshot

8. GET /services/{name}/logs
   - Handler: get_service_logs()
   - Query: ?lines=100&filter=ERROR
   - Response: LogsResponse { lines: Vec<LogLine>, total_lines, truncated }
   - Purpose: Retrieve service logs with filtering
