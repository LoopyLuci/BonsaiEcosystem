### 3. Core Architecture: The Proxy‑Based OAuth2 Login System

The desktop app contains **no Google client ID or secret**. Instead, a free Cloudflare Worker acts as a token exchange proxy.

**Authentication flow:**
1. User clicks **Login** in the GUI.
2. The app starts a temporary `aiohttp` server on a random `localhost` port.
3. The app opens the user’s browser to `https://<proxy>/auth/start?local_port=<port>&provider=google`.
4. The Cloudflare Worker (proxy) redirects to Google’s OAuth2 consent page using its own stored `client_id` and `client_secret`.
5. After the user grants permission, Google redirects back to the proxy.
6. The proxy exchanges the authorization code for access + refresh tokens.
7. The proxy redirects the browser to `http://localhost:<port>/callback?access_token=...`.
8. The local `aiohttp` server receives the tokens, stores them in an **encrypted SQLCipher database**, and the GUI refreshes.

**Advantages:**
- Zero credentials in the distributed app.
- If credentials need to be rotated, only the Cloudflare Worker is updated—no app rebuilds required.
- Uses PKCE, state parameter, and security headers for robust defense.

### 4. Key Files and Their Complete Code

#### 4.1 `proxy-worker.js` (Repository Root)
The hardened Cloudflare Worker that handles the OAuth dance. It reads `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` from Cloudflare Dashboard environment variables (`env`).

```javascript
/*
 * proxy-worker.js
 * Project: FTDaemon OAuth Proxy
 * Description: Hardened Edge Proxy for Google OAuth 2.0 PKCE.
 * 
 * SECURITY NOTE: Deploy using Cloudflare Dashboard Variables for
 * GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET.
 */

const REDIRECT_PATH = '/auth/callback';

const SECURITY_HEADERS = {
  'Content-Security-Policy': "default-src 'self'; frame-ancestors 'none'",
  'Strict-Transport-Security': 'max-age=63072000; includeSubDomains; preload',
  'X-Content-Type-Options': 'nosniff',
  'X-Frame-Options': 'DENY',
  'Referrer-Policy': 'no-referrer'
};

function addSecurityHeaders(response) {
  const newResponse = new Response(response.body, response);
  Object.entries(SECURITY_HEADERS).forEach(([key, value]) => {
    newResponse.headers.set(key, value);
  });
  return newResponse;
}

async function handleAuthStart(request, env, url) {
  const localPort = url.searchParams.get('local_port');
  if (!localPort) {
    return addSecurityHeaders(new Response('Missing local_port', { status: 400 }));
  }

  const codeVerifier = crypto.randomUUID() + crypto.randomUUID();
  const encoder = new TextEncoder();
  const data = encoder.encode(codeVerifier);
  const hash = await crypto.subtle.digest('SHA-256', data);
  const codeChallenge = btoa(String.fromCharCode(...new Uint8Array(hash)))
    .replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '');

  const state = crypto.randomUUID();
  const expires = Date.now() + 5 * 60 * 1000;
  
  globalThis.verifiers = globalThis.verifiers || new Map();
  globalThis.verifiers.set(state, { codeVerifier, localPort, expires });

  const googleAuthUrl = new URL('https://accounts.google.com/o/oauth2/v2/auth');
  googleAuthUrl.searchParams.set('client_id', env.GOOGLE_CLIENT_ID);
  googleAuthUrl.searchParams.set('redirect_uri', url.origin + REDIRECT_PATH);
  googleAuthUrl.searchParams.set('response_type', 'code');
  googleAuthUrl.searchParams.set('scope', 'https://www.googleapis.com/auth/drive.file');
  googleAuthUrl.searchParams.set('state', state);
  googleAuthUrl.searchParams.set('code_challenge', codeChallenge);
  googleAuthUrl.searchParams.set('code_challenge_method', 'S256');
  googleAuthUrl.searchParams.set('access_type', 'offline');
  googleAuthUrl.searchParams.set('prompt', 'consent');

  return addSecurityHeaders(Response.redirect(googleAuthUrl.href, 302));
}

async function handleAuthCallback(request, env, url) {
  const code = url.searchParams.get('code');
  const state = url.searchParams.get('state');

  if (!code || !state) {
    return addSecurityHeaders(new Response('Invalid parameters', { status: 400 }));
  }

  const entry = globalThis.verifiers?.get(state);
  if (!entry || entry.expires < Date.now()) {
    return addSecurityHeaders(new Response('State not found or expired', { status: 400 }));
  }
  globalThis.verifiers.delete(state);

  const tokenResponse = await fetch('https://oauth2.googleapis.com/token', {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: new URLSearchParams({
      client_id: env.GOOGLE_CLIENT_ID,
      client_secret: env.GOOGLE_CLIENT_SECRET,
      code,
      grant_type: 'authorization_code',
      redirect_uri: url.origin + REDIRECT_PATH,
      code_verifier: entry.codeVerifier,
    }),
  });

  if (!tokenResponse.ok) {
    const err = await tokenResponse.text();
    return addSecurityHeaders(new Response(`Token exchange failed: ${err}`, { status: 502 }));
  }

  const tokens = await tokenResponse.json();
  const localUrl = new URL(`http://localhost:${entry.localPort}/callback`);
  
  localUrl.searchParams.set('access_token', tokens.access_token);
  localUrl.searchParams.set('refresh_token', tokens.refresh_token || '');
  localUrl.searchParams.set('expires_in', tokens.expires_in || 3600);
  localUrl.searchParams.set('scope', tokens.scope || '');
  localUrl.searchParams.set('token_type', tokens.token_type || 'Bearer');

  return addSecurityHeaders(Response.redirect(localUrl.href, 302));
}

export default {
  async fetch(request, env) {
    const url = new URL(request.url);
    if (url.pathname === '/auth/start') {
      return handleAuthStart(request, env, url);
    } else if (url.pathname === REDIRECT_PATH) {
      return handleAuthCallback(request, env, url);
    }
    return addSecurityHeaders(new Response('Not found', { status: 404 }));
  },
};
```

#### 4.4 `src/core/cloud/oauth_flows.py` (Complete)

```python
import asyncio
import socket
import webbrowser
import logging
from urllib.parse import parse_qs
from aiohttp import web
from PySide6.QtCore import QThread, Signal

from src.core.utils.constants import OAUTH_PROXY_URL

logger = logging.getLogger(__name__)


class OAuthFlow:
    """Proxy‑based OAuth flow for desktop apps."""

    def __init__(self, token_manager):
        self.token_manager = token_manager

    async def authorize(self, provider):
        # Find a free port
        s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        s.bind(('', 0))
        port = s.getsockname()[1]
        s.close()

        auth_code_future = asyncio.Future()

        async def handle_callback(request):
            query = parse_qs(request.rel_url.query)
            access_token = query.get('access_token', [None])[0]
            refresh_token = query.get('refresh_token', [None])[0]
            expires_in = query.get('expires_in', [None])[0]
            scope = query.get('scope', [None])[0]
            token_type = query.get('token_type', [None])[0]

            if not access_token:
                auth_code_future.set_exception(Exception('No access token received'))
                return web.Response(text='Login failed: missing token')

            self.token_manager.store_token(provider, {
                'access_token': access_token,
                'refresh_token': refresh_token or '',
                'token_type': token_type or 'Bearer',
                'expires_at': int(expires_in or 3600),
                'scope': scope or '',
                'meta': {}
            })
            auth_code_future.set_result(True)
            return web.Response(text='Login successful! You may close this window.')

        app = web.Application()
        app.router.add_get('/callback', handle_callback)
        runner = web.AppRunner(app)
        await runner.setup()
        site = web.TCPSite(runner, 'localhost', port)
        await site.start()

        proxy_url = f"{OAUTH_PROXY_URL}/auth/start?local_port={port}&provider=google"
        webbrowser.open(proxy_url)
        logger.info("Opened browser for OAuth via proxy")
        return await auth_code_future


class OAuthLoginWorker(QThread):
    finished = Signal(bool, str)   # success, error message

    def __init__(self, flow, provider_name):
        super().__init__()
        self.flow = flow
        self.provider_name = provider_name

    def run(self):
        try:
            asyncio.run(self.flow.authorize(self.provider_name))
            self.finished.emit(True, '')
        except Exception as e:
            self.finished.emit(False, str(e))
```

```python
class TokenManager:
    def __init__(self, db_ops):
        self.db_ops = db_ops

    def store_token(self, provider, token_data):
        self.db_ops.store_cloud_token(provider, **token_data)

    def get_token(self, provider):
        row = self.db_ops.get_cloud_token(provider)
        if row:
```

### 5. Deployment Guide (Step‑by‑Step)

#### 5.1 Google Cloud Console Setup

1. Go to [Google Cloud Console](https://console.cloud.google.com/).
2. Create a new project (e.g., `FTDaemon`).
3. Enable the **Google Drive API** (APIs & Services → Library).
4. Configure the OAuth consent screen:
   - User Type: **External**.
   - App name: `FTDaemon`.
   - Scopes: `https://www.googleapis.com/auth/drive.file` (non‑sensitive).
   - Add your email as a test user.
5. Create an OAuth 2.0 Client ID:
   - Application type: **Web application** (because the proxy is the web‑facing part).
   - Authorized redirect URI: `https://<your-worker-url>/auth/callback` (set after worker deployment).

#### 5.2 Deploy the Cloudflare Worker

1. Install Wrangler CLI: `npm install -g wrangler`.
2. Log in: `wrangler login`.
3. Set secrets (from Google Cloud Console Client ID and Secret):
   ```bash
   wrangler secret put GOOGLE_CLIENT_ID
   wrangler secret put GOOGLE_CLIENT_SECRET
   ```
4. Ensure `proxy-worker.js` and `wrangler.toml` are in the repo root.
5. Deploy: `wrangler deploy`.
6. You'll receive a URL like `https://ftdaemon-auth-proxy.<subdomain>.workers.dev`.
7. Go back to Google Cloud Console and add `https://ftdaemon-auth-proxy.<subdomain>.workers.dev/auth/callback` as the authorized redirect URI.

#### 5.3 Update the Application

1. Edit `src/core/utils/constants.py` and set:
   ```python
   OAUTH_PROXY_URL = "https://ftdaemon-auth-proxy.<subdomain>.workers.dev"
   ```
2. Rebuild the NSIS installer (or re‑run `install-windows.bat`):
   ```bash
   makensis .\installers\install-windows.nsi
   ```
3. Launch the GUI using the root `FTDaemon.bat` or the Start Menu shortcut.

When scanning a user‑selected directory, FTDaemon spawns a Sanctum vault with read‑only capability to that directory. This prevents a bug or malicious file from corrupting the rest of the system.

### 2.5 Universe for Audit Trail

Every operation (copy, delete, move, upload, download) is logged to the local Universe (immutable audit log) and can optionally be synced to a remote cluster. Users can replay any session to see exactly what happened.

### 2.6 Bonsai Buddy & Omni‑Bot Integration

- **Bonsai Buddy** (mobile app) connects to FTDaemon via the same P2P fabric. Users can start a backup job, view progress, and receive notifications from their phone.

**How TransferDaemon is used:**

1. **Device Discovery** – FTDaemon registers itself with the local TransferDaemon node, receiving a cryptographic identity. The GUI displays all other FTDaemon instances on the same mesh (e.g., “Living Room Laptop”, “Office Desktop”). No IP addresses or manual configuration needed.

2. **File Transfer** – When a user selects files to send to another device, FTDaemon chunks them (FastCDC), deduplicates against the local CAS, and sends only new chunks via TransferDaemon’s `send` API. The receiver reassembles the file and verifies integrity with BLAKE3.

3. **Cloud Upload** – For cloud providers, FTDaemon uses the standard HTTP APIs (Google Drive, OneDrive, Dropbox) or SFTP/WebDAV. 4. **Resume & Reliability** – TransferDaemon’s session persistence ensures that interrupted transfers resume from the last acknowledged chunk. FTDaemon’s job manager serialises the job state to the local database; even after an app restart, pending transfers continue.

5. **Offline Operation** – When no network is available, FTDaemon operates purely locally: scanning, hashing, dedup, and audit logging all work without connectivity. When the mesh becomes available, queued P2P transfers are automatically resumed.

### 2.3 Content‑Addressed Deduplication (CAS)

FTDaemon’s deduplication engine uses the **Content‑Addressed Storage (CAS)** service (already part of the Omnisystem) as its backend. Every file, regardless of type or location, is split into variable‑size chunks using the FastCDC algorithm. Each chunk is hashed with BLAKE3 and stored in the local CAS.

**Deduplication workflow:**

1. User adds a directory to scan.
2. FTDaemon spawns a **Sanctum vault** with read‑only capability to that directory (see 2.4).
3. The vault walks the directory tree, reading each file in 64 KB chunks.
4. Each chunk is hashed (xxhash for fast matching, BLAKE3 for collision resistance).
5. The chunk hash is looked up in the local CAS. If it already exists, a reference is added; if not, the chunk is compressed (UCE) and stored.
6. A file manifest (ordered list of chunk hashes) is stored in the local database.
7. When the user requests a backup, only **new chunks** (not present in the destination’s CAS) are transferred.

This approach gives:
- **Global dedup** across all backups, all devices, all cloud providers.
- **Incremental backups** – only changed chunks are transferred.
- **Instant file integrity verification** – the manifest acts as a Merkle tree.

### 2.4 Sanctum for Sandboxed Scanning

File scanning is the most dangerous operation FTDaemon performs: it must open and read arbitrary files from the user’s filesystem. To prevent a maliciously crafted file (e.g., a zip bomb, a symlink escape, or a zero‑day vulnerability in the hashing library) from compromising the host, FTDaemon runs **all scanning operations inside a Sanctum vault**.

**Sandbox setup:**

```titan
// FTDaemon core (Rust)
let scan_cap = capability::derive_read_only(root_path)?;
let vault = SanctumVault::create(
    "ftdaemon-scanner",
    vec![scan_cap, Capability::Cpu(1), Capability::Memory(256)],
    scanner_binary_hash,
)?;
vault.start();
```

The scanner vault:
- Has read‑only access to the user‑selected directory (and nothing else).
- Has no network access.
- Has a hard memory limit (256 MB) – a zip bomb that decompresses to terabytes is safely contained.
- Is destroyed after the scan completes, and its resources are reclaimed.

If the scanner vault crashes (e.g., due to a bug in the chunking library), the main FTDaemon process is unaffected. The job is marked as failed, and the user is notified.

### 2.5 Universe for Immutable Audit Logging

Every operation performed by FTDaemon – scan, hash, dedup, upload, download, delete, rename – is logged to the **Universe** service (immutable, content‑addressed audit log). Each log entry is a structured JSON object:

```json
{
  "timestamp": 1718123456,
  "operation": "upload",
  "file_path": "/home/user/Documents/report.pdf",
  "chunk_hashes": ["b3:a1b2...", "b3:c3d4..."],
  "provider": "google-drive",
  "remote_path": "/backups/2024/report.pdf",
  "user_id": "user-1234",
  "session_id": "sess-5678"
}
```

**Benefits:**
- **Full traceability** – every change to the file system is recorded.
- **Undo support** – the GUI can replay the log backwards to undo an operation.
- **Tamper‑proof** – each log entry is hashed; the chain is verifiable.
- **Replay for debugging** – the UVM can replay a backup session to reproduce a bug.

The Universe service is already integrated with the Omnisystem’s observability stack; FTDaemon uses the same client library.

```rust
//! FTDaemon Core – Deduplication, P2P, audit, and FFI bindings.

pub mod dedup;
pub mod p2p;
pub mod audit;
pub mod job;

use std::ffi::{CStr, CString};
use std::os::raw::c_char;
use std::sync::Arc;
use parking_lot::RwLock;

// Global state (for FFI simplicity)
static DEDUP_ENGINE: RwLock<Option<Arc<dedup::DedupEngine>>> = RwLock::new(None);
static P2P_CLIENT: RwLock<Option<Arc<p2p::P2pClient>>> = RwLock::new(None);

/// Initialize the core with a CAS path and TransferDaemon config.
#[no_mangle]
pub extern "C" fn ftdaemon_init(cas_path: *const c_char, config_json: *const c_char) -> i32 {
    let path = unsafe { CStr::from_ptr(cas_path).to_str().unwrap() };
    let config = unsafe { CStr::from_ptr(config_json).to_str().unwrap() };
    match dedup::DedupEngine::new(path) {
        Ok(engine) => { *DEDUP_ENGINE.write() = Some(Arc::new(engine)); }
        Err(e) => { eprintln!("Dedup init error: {}", e); return -1; }
    }
    match p2p::P2pClient::new(config) {
        Ok(client) => { *P2P_CLIENT.write() = Some(Arc::new(client)); }
        Err(e) => { eprintln!("P2P init error: {}", e); return -1; }
    }
    0
}

/// Scan a directory, deduplicate its files, return a JSON summary.
#[no_mangle]
pub extern "C" fn ftdaemon_scan_directory(dir_path: *const c_char) -> *mut c_char {
    let path = unsafe { CStr::from_ptr(dir_path).to_str().unwrap() };
    let engine = DEDUP_ENGINE.read().as_ref().unwrap().clone();
    let result = engine.scan_directory(path);
    let json = serde_json::to_string(&result).unwrap();
    CString::new(json).unwrap().into_raw()
}

/// Start a backup job to a remote target (cloud or peer).
#[no_mangle]
pub extern "C" fn ftdaemon_backup(job_json: *const c_char) -> *mut c_char {
    let job_str = unsafe { CStr::from_ptr(job_json).to_str().unwrap() };
    let job: job::BackupJob = serde_json::from_str(job_str).unwrap();
    let p2p = P2P_CLIENT.read().as_ref().unwrap().clone();
    let result = p2p.backup(job);
    CString::new(serde_json::to_string(&result).unwrap()).unwrap().into_raw()
}

/// Free a string returned by the library.
#[no_mangle]
pub extern "C" fn ftdaemon_free_string(ptr: *mut c_char) {
    if !ptr.is_null() {
        unsafe { drop(CString::from_raw(ptr)); }
    }
}
```

```rust
//! Content‑addressed deduplication using CAS.

use cas::CasClient;
use fastcdc::fastcdc;
use blake3::Hash;
use serde::{Serialize, Deserialize};
use std::collections::HashMap;
use std::fs;
use std::path::Path;
use std::sync::Arc;
use parking_lot::RwLock;

pub struct DedupEngine {
    cas: CasClient,
    chunk_cache: RwLock<HashMap<Hash, Vec<u8>>>,
}

#[derive(Serialize, Deserialize)]
pub struct ScanResult {
    pub total_files: usize,
    pub unique_chunks: usize,
    pub total_bytes: u64,
    pub dedup_bytes: u64,
}

impl DedupEngine {
    pub fn new(cas_path: &str) -> anyhow::Result<Self> {
        let cas = CasClient::new(cas_path)?;
        Ok(Self { cas, chunk_cache: RwLock::new(HashMap::new()) })
    }

    pub fn scan_directory(&self, path: &str) -> ScanResult {
        let mut total_files = 0;
        let mut unique_chunks = 0;
        let mut total_bytes = 0;
        let mut dedup_bytes = 0;

        for entry in walkdir::WalkDir::new(path).into_iter().filter_map(|e| e.ok()) {
            if entry.file_type().is_file() {
                total_files += 1;
                let data = fs::read(entry.path()).unwrap();
                total_bytes += data.len() as u64;
                let chunks = fastcdc(&data, 8192, 16384, 32768);
                let mut chunk_hashes = Vec::new();
                for chunk in chunks {
                    let hash = blake3::hash(&chunk.data);
                    if !self.cas.exists(&hash) {
                        unique_chunks += 1;
                        self.cas.put(&hash, &chunk.data).unwrap();
                    } else {
                        dedup_bytes += chunk.data.len() as u64;
                    }
                    chunk_hashes.push(hash);
                }
                // store file manifest (list of chunk hashes) in CAS
                let manifest = serde_json::to_vec(&chunk_hashes).unwrap();
                self.cas.put(&blake3::hash(&manifest), &manifest).unwrap();
            }
        }
        ScanResult { total_files, unique_chunks, total_bytes, dedup_bytes }
    }
}
```

```rust
//! P2P client using TransferDaemon.

use p2p_core::{Session, Transfer, FileChunk};
use serde::{Serialize, Deserialize};
use std::sync::Arc;

pub struct P2pClient {
    session: Arc<Session>,
}

impl P2pClient {
    pub fn new(config_json: &str) -> anyhow::Result<Self> {
        let session = Session::connect(config_json)?;
        Ok(Self { session: Arc::new(session) })
    }

    pub fn backup(&self, job: BackupJob) -> BackupResult {
        let mut result = BackupResult::default();
        for file in job.files {
            let transfer = Transfer::new(&file.path);
            let outcome = self.session.send_transfer(&job.target, transfer);
            result.files_processed += 1;
            if outcome.is_ok() { result.files_succeeded += 1; }
            else { result.files_failed += 1; }
        }
        result
    }
}

#[derive(Serialize, Deserialize)]
pub struct BackupJob {
    pub target: String,      // peer ID or cloud provider name
    pub files: Vec<FileSpec>,
}

#[derive(Serialize, Deserialize)]
pub struct FileSpec {
    pub path: String,
}

#[derive(Serialize, Deserialize, Default)]
pub struct BackupResult {
    pub files_processed: u32,
    pub files_succeeded: u32,
    pub files_failed: u32,
}
```

```python
import socket
import webbrowser
import logging
from urllib.parse import parse_qs
from aiohttp import web
from PySide6.QtCore import QThread, Signal

from src.core.utils.constants import OAUTH_PROXY_URL

logger = logging.getLogger(__name__)

class OAuthFlow:
    def __init__(self, token_manager):
        self.token_manager = token_manager

    async def authorize(self, provider):
        s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        s.bind(('', 0))
        port = s.getsockname()[1]
        s.close()

        auth_future = asyncio.Future()

        async def handle_callback(request):
            query = parse_qs(request.rel_url.query)
            access_token = query.get('access_token', [None])[0]
            refresh_token = query.get('refresh_token', [None])[0]
            expires_in = query.get('expires_in', [None])[0]
            if not access_token:
                auth_future.set_exception(Exception('No token received'))
                return web.Response(text='Failed')
            self.token_manager.store_token(provider, {
                'access_token': access_token,
                'refresh_token': refresh_token or '',
                'expires_at': expires_in or 3600,
            })
            auth_future.set_result(True)
            return web.Response(text='Login successful! You may close this window.')

        app = web.Application()
        app.router.add_get('/callback', handle_callback)
        runner = web.AppRunner(app)
        await runner.setup()
        site = web.TCPSite(runner, 'localhost', port)
        await site.start()

        proxy_url = f"{OAUTH_PROXY_URL}/auth/start?local_port={port}&provider={provider}"
        webbrowser.open(proxy_url)
        logger.info("Browser opened for OAuth via proxy")
        return await auth_future
```

```javascript
// Deploy to Cloudflare Workers
const REDIRECT_PATH = '/auth/callback';

const SECURITY_HEADERS = {
  'Content-Security-Policy': "default-src 'self'; frame-ancestors 'none'",
  'Strict-Transport-Security': 'max-age=63072000; includeSubDomains; preload',
  'X-Content-Type-Options': 'nosniff',
  'X-Frame-Options': 'DENY',
  'Referrer-Policy': 'no-referrer'
};

function addSecurityHeaders(response) {
  const newResponse = new Response(response.body, response);
  Object.entries(SECURITY_HEADERS).forEach(([k,v]) => newResponse.headers.set(k,v));
  return newResponse;
}

async function handleAuthStart(request, env, url) {
  const localPort = url.searchParams.get('local_port');
  if (!localPort) return addSecurityHeaders(new Response('Missing local_port', { status: 400 }));

  const codeVerifier = crypto.randomUUID() + crypto.randomUUID();
  const encoder = new TextEncoder();
  const data = encoder.encode(codeVerifier);
  const hash = await crypto.subtle.digest('SHA-256', data);
  const codeChallenge = btoa(String.fromCharCode(...new Uint8Array(hash)))
    .replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '');

  const state = crypto.randomUUID();
  const expires = Date.now() + 5 * 60 * 1000;
  globalThis.verifiers = globalThis.verifiers || new Map();
  globalThis.verifiers.set(state, { codeVerifier, localPort, expires });

  const googleAuthUrl = new URL('https://accounts.google.com/o/oauth2/v2/auth');
  googleAuthUrl.searchParams.set('client_id', env.GOOGLE_CLIENT_ID);
  googleAuthUrl.searchParams.set('redirect_uri', url.origin + REDIRECT_PATH);
  googleAuthUrl.searchParams.set('response_type', 'code');
  googleAuthUrl.searchParams.set('scope', 'https://www.googleapis.com/auth/drive.file');
  googleAuthUrl.searchParams.set('state', state);
  googleAuthUrl.searchParams.set('code_challenge', codeChallenge);
  googleAuthUrl.searchParams.set('code_challenge_method', 'S256');
  googleAuthUrl.searchParams.set('access_type', 'offline');
  googleAuthUrl.searchParams.set('prompt', 'consent');

  return addSecurityHeaders(Response.redirect(googleAuthUrl.href, 302));
}

async function handleAuthCallback(request, env, url) {
  const code = url.searchParams.get('code');
  const state = url.searchParams.get('state');
  if (!code || !state) return addSecurityHeaders(new Response('Missing parameters', { status: 400 }));

  const entry = globalThis.verifiers?.get(state);
  if (!entry || entry.expires < Date.now()) return addSecurityHeaders(new Response('Invalid state', { status: 400 }));
  globalThis.verifiers.delete(state);

  const tokenRes = await fetch('https://oauth2.googleapis.com/token', {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: new URLSearchParams({
      client_id: env.GOOGLE_CLIENT_ID,
      client_secret: env.GOOGLE_CLIENT_SECRET,
      code,
      grant_type: 'authorization_code',
      redirect_uri: url.origin + REDIRECT_PATH,
      code_verifier: entry.codeVerifier,
    }),
  });
  if (!tokenRes.ok) return addSecurityHeaders(new Response('Token exchange failed', { status: 502 }));

  const tokens = await tokenRes.json();
  const localUrl = new URL(`http://localhost:${entry.localPort}/callback`);
  localUrl.searchParams.set('access_token', tokens.access_token);
  localUrl.searchParams.set('refresh_token', tokens.refresh_token || '');
  localUrl.searchParams.set('expires_in', tokens.expires_in || 3600);
  return addSecurityHeaders(Response.redirect(localUrl.href, 302));
}

export default {
  async fetch(request, env) {
    const url = new URL(request.url);
    if (url.pathname === '/auth/start') return handleAuthStart(request, env, url);
    if (url.pathname === REDIRECT_PATH) return handleAuthCallback(request, env, url);
    return addSecurityHeaders(new Response('Not found', { status: 404 }));
  },
};
```

impl JobManager {
    pub fn new(max_retries: u32) -> Self {
        Self { queue: Arc::new(Mutex::new(VecDeque::new())), max_retries }
    }

    /// Enqueue a new job. Returns its ID.
    pub fn enqueue(&self, mut job: BackupJob) -> String {
        job.id = uuid::Uuid::new_v4().to_string();
        job.state = JobState::Queued;
        self.queue.lock().push_back(job.clone());
        job.id
    }

    /// Get the next ready job (Queued or Failed with retries remaining).
    pub fn dequeue(&self) -> Option<BackupJob> {
        let mut q = self.queue.lock();
        q.iter_mut().find(|j| matches!(j.state, JobState::Queued)
            || matches!(&j.state, JobState::Failed { retries } if *retries < self.max_retries))
            .map(|j| { j.state = JobState::Running; j.clone() })
    }

    /// Mark a job as completed.
    pub fn complete(&self, id: &str) {
        if let Some(job) = self.queue.lock().iter_mut().find(|j| j.id == id) {
            job.state = JobState::Completed;
        }
    }

    /// Mark a job as failed; increment retry count.
    pub fn fail(&self, id: &str) {
        if let Some(job) = self.queue.lock().iter_mut().find(|j| j.id == id) {
            let retries = match &job.state { JobState::Failed { retries } => *retries + 1, _ => 1 };
            job.state = JobState::Failed { retries };
        }
    }

    pub fn list(&self) -> Vec<BackupJob> {
        self.queue.lock().iter().cloned().collect()
    }
}

```python
import json, ctypes, asyncio
from pathlib import Path
from PySide6.QtWidgets import (QDialog, QVBoxLayout, QHBoxLayout, QLabel,
                               QLineEdit, QPushButton, QProgressBar, QFileDialog,
                               QComboBox, QMessageBox)
from PySide6.QtCore import QThread, Signal

lib = ctypes.CDLL("ftdaemon_core.dll" if __import__('sys').platform == "win32" else "libftdaemon.so")
lib.ftdaemon_backup.argtypes = [ctypes.c_char_p]
lib.ftdaemon_backup.restype = ctypes.c_void_p
lib.ftdaemon_free_string.argtypes = [ctypes.c_void_p]

class BackupWorker(QThread):
    progress = Signal(int)
    finished = Signal(bool, str)

    def __init__(self, job_json):
        super().__init__()
        self.job_json = job_json

    def run(self):
        try:
            ptr = lib.ftdaemon_backup(self.job_json.encode())
            result = ctypes.string_at(ptr).decode()
            lib.ftdaemon_free_string(ptr)
            self.finished.emit(True, result)
        except Exception as e:
            self.finished.emit(False, str(e))

class BackupDialog(QDialog):
    def __init__(self, parent=None):
        super().__init__(parent)
        self.setWindowTitle("New Backup Job")
        self.resize(500, 300)
        layout = QVBoxLayout(self)

        layout.addWidget(QLabel("Source Directory or File:"))
        self.source_edit = QLineEdit()
        browse_btn = QPushButton("Browse...")
        browse_btn.clicked.connect(self._browse_source)
        src_layout = QHBoxLayout()
        src_layout.addWidget(self.source_edit)
        src_layout.addWidget(browse_btn)
        layout.addLayout(src_layout)

        layout.addWidget(QLabel("Destination:"))
        self.dest_combo = QComboBox()
        self.dest_combo.addItems(["Google Drive", "OneDrive", "Dropbox", "Local Peer"])
        layout.addWidget(self.dest_combo)

        self.progress_bar = QProgressBar()
        self.progress_bar.setVisible(False)
        layout.addWidget(self.progress_bar)

        self.start_btn = QPushButton("Start Backup")
        self.start_btn.clicked.connect(self._start_backup)
        layout.addWidget(self.start_btn)

    def _browse_source(self):
        path = QFileDialog.getExistingDirectory(self, "Select Source Directory")
        if path:
            self.source_edit.setText(path)

    def _start_backup(self):
        source = self.source_edit.text()
        if not source:
            QMessageBox.warning(self, "Error", "Please select a source directory.")
            return
        dest = self.dest_combo.currentText().lower().replace(" ", "-")
        job = {"source": source, "target": dest, "remote_path": f"/backups/{Path(source).name}"}
        self.progress_bar.setVisible(True)
        self.start_btn.setEnabled(False)
        self.worker = BackupWorker(json.dumps(job))
        self.worker.finished.connect(self._on_finished)
        self.worker.start()

    def _on_finished(self, success, result):
        self.progress_bar.setVisible(False)
        self.start_btn.setEnabled(True)
        if success:
            QMessageBox.information(self, "Backup Complete", f"Backup finished:\n{result}")
        else:
            QMessageBox.critical(self, "Backup Failed", result)
```

| Principle | Implementation |
|-----------|----------------|
| **Each language does what it does best** | No language is forced to write code outside its paradigm. Connectors translate boundaries. |
| **Connectors are first‑class UMS modules** | Every connector is signed, versioned, hot‑reloadable, and formally verified. |
| **Titan is the systems glue** | High‑performance, low‑overhead bridge between languages that share a similar runtime (e.g., Rust ↔ C++ ↔ Zig). |
| **Sylva is the dynamic glue** | Scripting‑level connectors for languages with dynamic runtimes (Python, Ruby, JavaScript). |
| **Aether is the distributed glue** | Connectors that involve remote nodes or actor‑based concurrency (Erlang, Elixir, Go). |
| **Axiom proves connector correctness** | Each connector has an accompanying proof that it preserves semantics and never leaks memory or capabilities. |
| **Connections Index = UMS + UPLD** | The UPLD already stores language specs; the Index adds connector metadata. |
| **Connections Router = Aether actor** | Resolves (src, tgt) → connector hash, loads it, and delegates calls. |

## 4. Connector Types (Built with Titan, Sylva, Aether, Axiom)

### 4.1 Titan Connector – For compiled, statically‑typed languages

Titan provides zero‑overhead FFI to any language that can produce a C ABI. The connector is a Titan library that:

- Translates the source language’s data structures to Universal IR (UIR).
- Calls the target language’s exported functions via the Universal ABI.
- Handles memory ownership (e.g., Rust’s ownership model is preserved).

**Example**: Rust (source) → C++ (target) connector.

### 4.2 Sylva Connector – For dynamic, interpreted languages

Sylva’s JIT and dynamic type system allow it to wrap languages like Python, JavaScript, or Ruby. The connector:

- Embeds the target language’s interpreter (e.g., CPython) as a Sanctum vault.
- Translates Sylva dynamic values to the target language’s objects.
- Uses the target language’s C API for high‑performance calls.

**Example**: Python (source) → JavaScript (target) connector for asset pipeline.

### 4.3 Aether Connector – For actor‑based or distributed languages

Aether’s location transparency and message passing make it ideal for connecting Erlang, Elixir, or Go. The connector:

- Spawns an Aether actor that represents a remote actor in the target language.
- Serialises messages to the target language’s format (e.g., Erlang terms).
- Handles supervision and failure propagation.

**Example**: Erlang (source) → Go (target) connector for service mesh.

### 4.4 Axiom Connector – For verified, proof‑carrying code

Axiom connectors are used when the connector itself must be formally proven correct. The connector:

- Contains an Axiom proof that the translation preserves semantics.
- Is extracted to Titan or Sylva for execution.
- Can be used as a bridge between any language and the Axiom proof checker.

**Example**: Prolog (source) → Axiom (target) connector for rule validation.

## 5. Connections Index – Structure and Management

The Connections Index is stored as a **UMS module** that contains a mapping from `(source_language, target_language, source_version, target_version)` to a **connector hash** (BLAKE3). The index itself is content‑addressed and signed.

### 5.1 Index Schema (Titan)

```titan
// connections/index.ti
pub struct ConnectorEntry {
    pub source_lang: String,
    pub target_lang: String,
    pub source_version: SemVer,
    pub target_version: SemVer,
    pub connector_hash: [u8; 32],   // BLAKE3 of connector UMS module
    pub proof_hash: Option<[u8; 32]>, // Axiom proof (if any)
    pub performance_score: f32,       // µs per call (lower is better)
    pub council_signature: [u8; 64],
}

pub struct ConnectionsIndex {
    pub entries: Vec<ConnectorEntry>,
    pub index_hash: [u8; 32],
}
```

### 5.2 Index Management

- **Addition**: When a new connector is built, its entry is added to the index and the index is re‑signed by the council.
- **Versioning**: Multiple versions of a connector can coexist; the router selects the one with the highest compatible version and best performance score.
- **Distribution**: The index is distributed via UMS (content‑addressed). All Omnisystem nodes automatically sync the latest index.

### 5.3 Index Query API (Aether actor)

```aether
actor IndexRouter {
    pub fn resolve(src: String, tgt: String, src_ver: SemVer, tgt_ver: SemVer) -> Option<ConnectorEntry> {
        // query the local index cache; if not found, fetch from UMS
    }
}
```

## 6. Connections Router – Runtime Resolution and Delegation

The router is a **hot‑reloadable Aether actor** that intercepts cross‑language calls and dynamically loads connectors.

### 6.1 Router State Machine

1. **Receive call** – from language A, with target language B and serialised arguments (as UIR terms).
2. **Resolve connector** – query the Index for the best matching connector entry.
3. **Load connector** – if not already in memory, fetch the UMS module, verify its signature, and spawn a Sanctum vault with the connector code.
4. **Delegate** – pass the arguments to the connector’s `call` function (via Universal ABI).
5. **Return result** – forward the result back to the caller.

### 6.2 Router Implementation Sketch (Aether)

```aether
actor ConnectionsRouter {
    var connector_cache: Map<(String,String), ConnectorHandle> = empty()

    on call(src: String, tgt: String, args: Term) -> Term {
        let key = (src, tgt)
        let handle = connector_cache.get(key)
        if handle == null {
            let entry = index.resolve(src, tgt, get_versions())
            if entry == null { return error("No connector found") }
            let connector = load_connector(entry.connector_hash)
            connector_cache.set(key, connector)
            handle = connector
        }
        return handle.call(args)
    }
}
```

### 6.3 Connector Caching and Hot‑Reload

- The cache stores a **generation counter** per connector. When a new version of a connector is published, the index is updated, and the router atomically swaps the old handle for the new one (using the same atomic generation mechanism as the kernel’s hot‑reload). In‑flight calls finish on the old version; new calls use the new version.

## 7. Building a Connector – Step‑by‑Step Example

Suppose we want a connector between **Rust (source)** and **Python (target)**.

### 7.1 Identify the Rust function to expose

```rust
// Rust: compute something fast
#[no_mangle]
pub extern "C" fn fast_algorithm(input: *const u8, len: usize) -> *mut u8 {
    // ...
}
```

### 7.2 Write the Titan connector

```titan
// connectors/rust_to_python.ti
module connectors::rust_to_python

use ums::UmsClient
use titan::ffi::call_extern

pub fn call_rust_function(args: Term) -> Term {
    let input_bytes = term::get_bytes(args);
    let result_ptr = call_extern("fast_algorithm", input_bytes.as_ptr(), input_bytes.len());
    let result_bytes = unsafe { Vec::from_raw_parts(result_ptr, len) };
    term::alloc_bytes(&result_bytes)
}
```

### 7.3 Register the connector in the Connections Index

```bash
# Build connector UMS module
build module create --name rust-to-python --path target/rust_to_python.so
# Add entry to index
connections index add --source rust --target python --connector-hash $(blake3 target/rust_to_python.so) --council-sign
```

### 7.4 Use from Python

```python
from omnisystem.router import call

result = call("rust", "fast_algorithm", b"hello")
```

The router automatically loads the connector and delegates.

## 8. Formal Verification of Connectors

Every connector intended for production use must be accompanied by an **Axiom proof** that the translation preserves semantics. The proof is stored as a separate UMS module and referenced in the Connections Index.

**Example proof for a connector that passes an integer from Python to Rust:**

```ax
theorem python_int_to_rust_i64_preserves_value :
  forall n: i64,
    let python_value = python_int(n)
    let rust_value = call_connector("rust_to_python", python_value)
    rust_value == n
```

The proof is checked at connector load time. If verification fails, the connector is rejected.

## 3. Connector Architecture and Types

Connectors are the universal translators. They are themselves UMS modules, written in the language best suited for the specific translation task. We categorise them by the language pair and the nature of the connection.

### 3.1 Connector Categories

| Type | Best Implementation Language | Use Case |
|------|------------------------------|----------|
| **Native/ABI** | Titan, C, Zig | Compile‑time, zero‑overhead (Rust ↔ C++, Zig ↔ Titan) |
| **Dynamic/Foreign** | Sylva, Python | Runtime bridging (Python ↔ JavaScript, Lua ↔ Titan) |
| **Actor/Distributed** | Aether, Erlang | Remote actor communication (Erlang ↔ Go, Elixir ↔ Aether) |
| **Proof‑carrying** | Axiom, Haskell | Verified translation (Prolog ↔ Axiom, Haskell ↔ Coq) |
| **Data‑serialisation** | Titan, C++ | High‑speed IPC via shared memory (Rust ↔ Titan, C# ↔ Titan) |
| **API‑gateway** | Sylva, Go | REST/WebSocket bridges (JavaScript ↔ any service) |

### 3.2 Connector Lifecycle

1. **Design** – Identify the source and target languages, and the data types to be translated.
2. **Implement** – Write the connector in the chosen language, using the Universal ABI (term heap) as the common data representation.
3. **Test** – Validate with the Universal Validation Mesh (UVM) using deterministic cross‑language test suites.
4. **Prove** – For critical connectors, write an Axiom proof of semantic preservation.
5. **Package** – Build as a UMS module, sign with the governance council.
6. **Register** – Add entry to the Connections Index.
7. **Deploy** – The Connections Router automatically loads it on demand.

### 3.3 Connector Implementation Example (Rust → Python)

The connector is a Titan library that links against CPython’s C API. It maps Rust’s `Vec<u8>` to a Python `bytes` object, invokes a Python function, and returns the result as a Titan term.

```titan
// connectors/rust_to_python.ti
module connectors::rust_to_python

use ffi::python::PythonRuntime
use term::{Term, alloc_string, get_string}

pub fn call_python_function(module: &str, func: &str, args: Term) -> Term {
    let py = PythonRuntime::get_instance();
    let module_obj = py.import(module);
    let func_obj = module_obj.get_attr(func);
    let py_args = term_to_py(args);
    let result = func_obj.call(py_args);
    py_to_term(result)
}

fn term_to_py(term: Term) -> PyObject { /* ... */ }
fn py_to_term(obj: PyObject) -> Term { /* ... */ }
```

This connector runs inside a Sanctum vault with the `PYTHON` capability (which grants access to the Python runtime). The Python interpreter itself runs in a separate vault for isolation.

## 4. Connections Index – The Central Registry

The **Connections Index** is a content‑addressed, signed UMS module that maps every supported (source, target, version) triple to the hash of the best‑available connector. It also stores metadata: performance score, Axiom proof hash, council signature.

### 4.1 Schema

```titan
// connections/index.ti
pub struct ConnectorEntry {
    pub source_lang: String,          // e.g., "rust"
    pub target_lang: String,          // e.g., "python"
    pub source_version: SemVer,       // min required version
    pub target_version: SemVer,
    pub connector_hash: [u8; 32],     // BLAKE3 of connector UMS module
    pub proof_hash: Option<[u8; 32]>, // Axiom proof of semantic preservation
    pub performance_score: f32,       // avg μs per call (lower = better)
    pub added: Timestamp,
    pub council_signatures: Vec<Signature>,
}

pub struct ConnectionsIndex {
    pub entries: Map<(String, String, SemVer, SemVer), ConnectorEntry>,
    pub index_hash: [u8; 32],
    pub version: u64,
}
```

### 4.2 Index Management

- The index is updated via a council vote (threshold signature). New connector entries are proposed, verified by the UVM, and appended.
- The index is content‑addressed; every node caches it locally and polls for updates via TransferDaemon.
- Multiple versions of a connector may coexist; the router selects the one with the highest confidence (formal proof + performance score).

### 4.3 Index Query Actor (Aether)

```aether
actor IndexService {
    var cache: Map<(String,String,SemVer,SemVer), ConnectorEntry> = empty()

    on resolve(src: String, tgt: String, src_ver: SemVer, tgt_ver: SemVer) -> Option<ConnectorEntry> {
        let key = (src, tgt, src_ver, tgt_ver)
        if cache.contains(key) { return cache.get(key) }
        // Fetch from UMS
        let entry = ums::get_index_entry(key)
        if entry != none { cache.set(key, entry) }
        return entry
    }
}
```

## 5. Connections Router – Runtime Cross‑Language Call Dispatcher

The **Connections Router** is an Aether actor that intercepts every cross‑language call and transparently loads, invokes, and manages connectors.

### 5.1 Call Flow

1. **Source language** (e.g., Rust) calls `router::call("python", "my_module", "my_func", args)`.
2. The Router receives the request as an Aether message.
3. It calls `IndexService::resolve("rust", "python", v1, v2)`.
4. If a connector is found, it checks its cache; if not loaded, it fetches the UMS module, verifies the signature, spawns a Sanctum vault with the connector, and stores the handle.
5. The Router serialises the arguments (as Universal ABI terms) and sends them to the connector’s vault via IPC.
6. The connector executes the target‑language function, receives the result, converts it back to a term, and returns it to the Router.
7. The Router forwards the result to the caller.

### 5.2 Router Actor (Aether)

```aether
actor ConnectionsRouter {
    var connector_cache: Map<(String,String), ConnectorHandle> = empty()
    var index: IndexService

    on call(src: String, tgt: String, src_ver: SemVer, tgt_ver: SemVer, module: String, func: String, args: Term) -> Term {
        let key = (src, tgt)
        let handle = connector_cache.get(key)
        if handle == null {
            let entry = index.resolve(src, tgt, src_ver, tgt_ver)
            if entry == null { return Term::Error("No connector") }
            let conn = load_connector(entry.connector_hash)
            connector_cache.set(key, conn)
            handle = conn
        }
        return handle.call(module, func, args)
    }
}
```

### 5.3 Hot‑Reload and Caching

- The cache maintains a generation counter per connector. When a new version is published, the router invalidates the cache entry; in‑flight calls complete on the old version. The next call loads the new connector atomically.
- Performance statistics (latency, throughput) are collected by the `observability` service and used to update the `performance_score` in the index.

## 6. Development Process for Language‑Specific Modules

Each of the 750+ languages contributes a **self‑contained module** that implements one or more well‑defined interfaces (e.g., `ImageFilter`, `Scheduler`, `LogParser`). The module is written in that language, compiled/interpreted within a Sanctum vault, and exposed via a connector.

### 6.1 Module Template

Every language module must provide:
- A **UMS manifest** declaring the language, version, capabilities required, and the exported API.
- An **entry point** (e.g., a shared library for compiled languages, or a script for interpreted ones).
- A **connector** (if one doesn’t already exist) that translates between the module’s native interface and the Universal ABI.

### 6.2 Community Contribution Pipeline

1. A contributor selects a language and a component from the “unclaimed” list in the Language Registry.
2. They fork the Omnisystem repository, implement the module and connector, and write UVM test plans.
3. They submit a pull request; CI runs the full cross‑language test matrix.
4. The council reviews the PR, checks the Axiom proof (if required), and upon approval, merges and publishes the new module and connector to UMS.

## 7. Formal Verification of Connectors

Critical connectors are formally verified with Axiom. The proof must show that for all valid inputs, the connector’s output is semantically equivalent to a direct call in the target language.

### 7.1 Proof Obligation Example

For the Rust‑to‑Python connector passing an integer:

```ax
theorem rust_to_python_int_roundtrip :
  forall n: i64,
    let py_val = rust_to_python_call("test_module", "identity", n)
    py_val == n
```

### 7.2 Proof Check at Load Time

When the Router loads a connector, it checks the proof certificate (if present). If the proof fails, the connector is rejected and an alert is sent to the observability service.
